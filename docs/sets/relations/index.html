Relations

(Content adapted from Critchlow & Eck)

In the Functions section, we saw that "mother of" is a functional relationship because every person has one and only one mother, but that "child of" is not a functional relationship, because a person can have no children or more than one child. However, the relationship expressed by "child of" is certainly one that we have a right to be interested in and one that we should be able to deal with mathematically.

There are many examples of relationships that are not functional relationships. The relationship that holds between two natural numbers nnn and mmm when n≤mn\le mn≤m is an example in mathematics. The relationship between a person and a book that that person has on loan from the library is another. Some relationships involve more than two entities, such as the relationship that associates a name, an address, and a phone number in an address book or the relationship that holds among three real numbers xxx, yyy, and zzz if x2+y2+z2=1x^2+y^2+z^2=1x2+y2+z2=1. Each of these relationships can be represented mathematically by what is called a "relation."

A relation on two sets, AAA and BBB, is defined to be a subset of A×BA\times BA×B. Since a function from AAA to BBB is defined, formally, as a subset of A×BA\times BA×B that satisfies certain properties, a function is a relation. However, relations are more general than functions, since any subset of A×BA\times BA×B is a relation. We also define a relation among three or more sets to be a subset of the cross product of those sets. In particular, a relation on AAA, BBB, and CCC is a subset of A×B×CA\times B\times CA×B×C.

For example, if PPP is the set of people and BBB is the set of books owned by a library, then we can define a relation R{\mathscr R}R on the sets PPP and BBB to be the set R={(p,b)∈P×B∣p has b out on loan}{\mathscr R}=\{(p,b)\in P\times B\mid p\text{ has }b\text{ out on loan}\}R={(p,b)∈P×B∣p has b out on loan}. The fact that a particular (p,b)∈R(p,b)\in{\mathscr R}(p,b)∈R is a fact about the world that the library will certainly want to keep track of. When a collection of facts about the world is stored on a computer, it is called a database. We'll see in the next section that relations are the most common means of representing data in databases.

If AAA is a set and R{\mathscr R}R is a relation on the sets AAA and AAA (that is, on two copies of AAA), then R\mathscr RR is said to be a binary relation on AAA. That is, a binary relation on the set AAA is a subset of A×AA\times AA×A. The relation consisting of all ordered pairs (c,p)(c,p)(c,p) of people such that ccc is a child of ppp is a binary relation on the set of people. The set {(n,m)∈N×N∣n≤m}\{(n,m)\in\N\times\N\mid n\le m\}{(n,m)∈N×N∣n≤m} is a binary relation on N\NN. Similarly, we define a ternary relation on a set AAA to be a subset of A×A×AA\times A\times AA×A×A. The set {(x,y,z)∈R×R×R∣x2+y2+z2=1}\{(x,y,z)\in\R\times\R\times\R\mid x^2+y^2+z^2=1\}{(x,y,z)∈R×R×R∣x2+y2+z2=1} is a ternary relation on R\RR. For complete generality, we can define an nnn-ary relation on AAA, for any positive integer nnn, to be a subset of A×A×⋯×AA\times A\times\dots\times AA×A×⋯×A, where AAA occurs nnn times in the cross product.

For the rest of this section, we will be working exclusively with binary relations. Suppose that R⊆A×A{\mathscr R}\subseteq A\times AR⊆A×A. That is, suppose that R{\mathscr R}R is a binary relation on a set AAA. If (a,b)∈R(a,b)\in{\mathscr R}(a,b)∈R, then we say that aaa is related to bbb by R{\mathscr R}R. Instead of writing "(a,b)∈R(a,b)\in {\mathscr R}(a,b)∈R", we will often write "a R ba\,{\mathscr R}\,baRb". This notation is used in analogy to the notation n≤mn\le mn≤m to express the relation that nnn is less than or equal to mmm. Remember that a R ba\,{\mathscr R}\,baRb is just an alternative way of writing (a,b)∈R(a,b)\in{\mathscr R}(a,b)∈R. In fact, we could consider the relation ≤\le≤ to be a set of ordered pairs and write (n,m)∈ ≤(n,m)\in\,\le(n,m)∈≤ in place of the notation n≤mn\le mn≤m.

Properties of Relations#

In many applications, attention is restricted to relations that satisfy some property or set of properties. (This is, of course, just what we do when we study functions.) We begin our discussion of binary relations by considering several important properties. In this discussion, let AAA be a set and let R{\mathscr R}R be a binary relation on AAA, that is, a subset of A×AA\times AA×A.

R\mathscr RR is said to be reflexive if ∀a∈A (a R a)\forall a\in A\,(a \,{\mathscr R}\, a)∀a∈A(aRa). That is, a binary relation on a set is reflexive if every element of the set is related to itself. This is true, for example, for the relation ≤\le≤ on the set N\NN, since n≤nn\le nn≤n for every n∈Nn\in\Nn∈N. On the other hand, it is not true for the relation <<< on N\NN, since, for example, the statement 17<1717<1717<17 is false.1

R\mathscr RR is called transitive if ∀a∈A, ∀b∈A, ∀c∈A ((a R b∧b R c)→(a R c))\forall a\in A,\,\forall b\in A,\,\forall c\in A\, \big((a\,{\mathscr R}\,b \land b\,{\mathscr R}\,c)\rightarrow (a\,{\mathscr R}\,c)\big)∀a∈A,∀b∈A,∀c∈A((aRb∧bRc)→(aRc)). Transitivity allows us to "chain together" two true statements a R ba\,{\mathscr R}\,baRb and b R cb\,{\mathscr R}\,cbRc, which are "linked" by the bbb that occurs in each statement, to deduce that a R ca\,{\mathscr R}\,caRc. For example, suppose PPP is the set of people, and define the relation C{\mathscr C}C on PPP such that x P yx\,{\mathscr P}\,yxPy if and only if xxx is a child of yyy. The relation P{\mathscr P}P is not transitive because the child of a child of a person is not a child of that person. Suppose, on the other hand, that we define a relation D{\mathscr D}D on PPP such that x D yx\,{\mathscr D}\,yxDy if and only if xxx is a descendent of yyy. Then DDD is a transitive relation on the set of people, since a descendent of a descendent of a person is a descendent of that person. That is, from the facts that Elizabeth is a descendent of Victoria and Victoria is a descendent of James, we can deduce that Elizabeth is a descendent of James. In the mathematical world, the relations ≤\le≤ and <<< on the set N\NN are both transitive.

R{\mathscr R}R is said to be symmetric if ∀a∈A, ∀b∈B (a R b→b R a)\forall a\in A,\,\forall b\in B\, (a\,{\mathscr R}\,b\rightarrow b\,{\mathscr R}\,a)∀a∈A,∀b∈B(aRb→bRa). That is, whenever aaa is related to bbb, it follows that bbb is related to aaa. The relation "is a first cousin of" on the set of people is symmetric, since whenever xxx is a first cousin of yyy, we have automatically that yyy is a first cousin of xxx. On the other hand, the "child of" relation is certainly not symmetric. The relation ≤\le≤ on N\NN is not symmetric. From the fact that n≤mn\le mn≤m, we cannot conclude that m≤nm\le nm≤n. It is true for some nnn and mmm in N\NN that n≤m→m≤nn\le m\rightarrow m\le nn≤m→m≤n, but it is not true for all nnn and mmm in N\NN.

Finally, R\mathscr RR is antisymmetric if ∀a∈A, ∀b∈B ((a R b∧b R a)→a=b)\forall a\in A,\,\forall b\in B\, \big((a\,{\mathscr R}\,b\land b\,{\mathscr R}\,a)\rightarrow a=b\big)∀a∈A,∀b∈B((aRb∧bRa)→a=b). The relation R\mathscr RR is antisymmetric if for any two distinct elements xxx and yyy of AAA, we can't have both x R yx\,{\mathscr R}\,yxRy and y R xy\,{\mathscr R}\,xyRx. The relation ≤\le≤ on N\NN is antisymmetric because from the facts that n≤mn\le mn≤m and m≤nm\le nm≤n, we can deduce that n=mn=mn=m. The relation "child of" on the set of people is antisymmetric since it's impossible to have both that xxx is a child of yyy and yyy is a child of xxx.

Ordering Relations#

There are a few combinations of properties that define particularly useful types of binary relations. The relation ≤\le≤ on the set N\NN is reflexive, antisymmetric, and transitive. These properties define what is called a partial order: A partial order on a set AAA is a binary relation on AAA that is reflexive, antisymmetric, and transitive.

Another example of a partial order is the subset relation, ⊆\subseteq⊆, on the power set of any set. If XXX is a set, then of course P(X){\mathscr P}(X)P(X) is a set in its own right, and ⊆\subseteq⊆ can be considered to be a binary relation on this set. Two elements AAA and BBB of P(X){\mathscr P}(X)P(X) are related by ⊆\subseteq⊆ if and only if A⊆BA\subseteq BA⊆B. This relation is reflexive since every set is a subset of itself. The fact that it is antisymmetric follows from the Set Equality Theorem. The fact that it is transitive was Exercise 11 in the Set Concepts section.

The ordering imposed on N\NN by ≤\le≤ has one important property that the ordering of subsets by ⊆\subseteq⊆ does not share. If nnn and mmm are natural numbers, then at least one of the statements n≤mn\le mn≤m and m≤nm\le nm≤n must be true. However, if AAA and BBB are subsets of a set XXX, it is certainly possible that both A⊆BA\subseteq BA⊆B and B⊆AB\subseteq AB⊆A are false. A binary relation R\mathscr RR on a set AAA is said to be a total order if it is a partial order and furthermore for any two elements aaa and bbb of AAA, either a R ba\,{\mathscr R}\,baRb or b R ab\,{\mathscr R}\,abRa. The relation ≤\le≤ on the set N\NN is a total order. The relation ⊆\subseteq⊆ on P(X){\mathscr P}(X)P(X) is not. (Note once again the slightly odd mathematical language: A total order is a kind of partial order—not, as you might expect, the opposite of a partial order.)

For another example of ordering, let LLL be the set of strings that can be made from lowercase letters. LLL contains both English words and nonsense strings such as "sxjja". There is a commonly used total order on the set LLL, namely alphabetical order.

Equivalence Relations and Partitions#

We'll approach another important kind of binary relation indirectly, through what might at first appear to be an unrelated idea. Let AAA be a set. A partition of AAA is defined to be a collection of non-empty subsets of AAA such that each pair of distinct subsets in the collection is disjoint and the union of all the subsets in the collection is AAA. A partition of AAA is just a division of all the elements of AAA into non-overlapping subsets. For example, the sets {1,2,6}\{1,2,6\}{1,2,6}, {3,7}\{3,7\}{3,7}, {4,5,8,10}\{4,5,8,10\}{4,5,8,10}, and {9}\{9\}{9} form a partition of the set {1,2,…,10}\{1,2,\dots,10\}{1,2,…,10}. Each element of {1,2,…,10}\{1,2,\dots,10\}{1,2,…,10} occurs in exactly one of the sets that make up the partition. As another example, we can partition the set of all people into two sets, the set of those who have taken Foundations and the set of those who have not. Biologists try to partition the set of all organisms into different species. Librarians try to partition books into various categories such as fiction, biography, and poetry. In the real world, classifying things into categories is an essential activity, although the boundaries between categories are not always well-defined. The abstract mathematical notion of a partition of a set models the real-world notion of classification. In the mathematical world, though, the categories are sets and the boundary between two categories is sharp.

In the real world, items are classified in the same category because they are related in some way. This leads us from partitions back to relations. Suppose that we have a partition of a set AAA. We can define a relation R\mathscr RR on AAA by declaring that for any aaa and bbb in AAA, a R ba\,{\mathscr R}\,baRb if and only if aaa and bbb are members of the same subset in the partition. That is, two elements of AAA are related if they are in the same category. It is clear that the relation defined in this way is reflexive, symmetric, and transitive.

An equivalence relation is defined to be a binary relation that is reflexive, symmetric, and transitive. Any relation defined, as above, from a partition is an equivalence relation. Conversely, we can show that any equivalence relation defines a partition. Suppose that R\mathscr RR is an equivalence relation on a set AAA. Let a∈Aa\in Aa∈A. We define the equivalence class of aaa under the equivalence relation R\mathscr RR to be the subset [a]R[a]_{\mathscr R}[a]R​ defined as [a]R={b∈A∣b R a}[a]_{\mathscr R}=\{b\in A\mid b\,{\mathscr R}\,a\}[a]R​={b∈A∣bRa}. That is, the equivalence class of aaa is the set of all elements of AAA that are related to aaa. In most cases, we'll assume that the relation in question is understood, and we'll write [a][a][a] instead of [a]R[a]_{\mathscr R}[a]R​. Note that each equivalence class is a subset of AAA. The following theorem shows that the collection of equivalence classes form a partition of AAA.

Theorem: Partitions Let AAA be a set and let R\mathscr RR be an equivalence relation on AAA. Then the collection of all equivalence classes under R\mathscr RR is a partition of AAA.

Proof: To show that a collection of subsets of AAA is a partition, we must show that each subset is non-empty, that the intersection of two distinct subsets is empty, and that the union of all the subsets is AAA.

If [a][a][a] is one of the equivalence classes, it is certainly non-empty, since a∈[a]a\in[a]a∈[a]. (This follows from the fact that R\mathscr RR is reflexive, and hence a R aa\,{\mathscr R}\,aaRa.) To show that AAA is the union of all the equivalence classes, we just have to show that each element of AAA is a member of one of the equivalence classes. Again, the fact that a∈[a]a\in[a]a∈[a] for each a∈Aa\in Aa∈A shows that this is true.

Finally, we have to show that the intersection of two distinct equivalence classes is empty. Suppose that aaa and bbb are elements of AAA and consider the equivalence classes [a][a][a] and [b][b][b]. We have to show that if [a]≠[b][a]\not=[b][a]​=[b], then [a]∩[b]=∅[a]\cap[b]=\emptyset[a]∩[b]=∅. Equivalently, we can show the converse: If [a]∩[b]≠∅[a]\cap[b]\not=\emptyset[a]∩[b]​=∅ then [a]=[b][a]=[b][a]=[b]. So, assume that [a]∩[b]≠∅[a]\cap[b]\not=\emptyset[a]∩[b]​=∅. Saying that a set is not empty just means that the set contains some element, so there must be an x∈Ax\in Ax∈A such that x∈[a]∩[b]x\in[a]\cap[b]x∈[a]∩[b]. Since x∈[a]x\in[a]x∈[a], x R ax\,{\mathscr R}\,axRa. Since R\mathscr RR is symmetric, we also have a R xa\,{\mathscr R}\,xaRx. Since x∈[b]x\in[b]x∈[b], x R bx\,{\mathscr R}\,bxRb. Since R\mathscr RR is transitive and since (a R x)∧(x R b)(a\,{\mathscr R}\,x)\land (x\,{\mathscr R}\,b)(aRx)∧(xRb), it follows that a R ba\,{\mathscr R}\,baRb.

Our object is to deduce that [a]=[b][a]=[b][a]=[b]. Since [a][a][a] and [b][b][b] are sets, they are equal if and only if [a]⊆[b][a]\subseteq[b][a]⊆[b] and [b]⊆[a][b]\subseteq[a][b]⊆[a]. To show that [a]⊆[b][a]\subseteq[b][a]⊆[b], let ccc be an arbitrary element of [a][a][a]. We must show that c∈[b]c\in[b]c∈[b]. Since c∈[a]c\in[a]c∈[a], we have that c R ac\,{\mathscr R}\,acRa. And we have already shown that a R ba\,{\mathscr R}\,baRb. From these two facts and the transitivity of R\mathscr RR, it follows that c R bc\,{\mathscr R}\,bcRb. By definition, this means that c∈[b]c\in[b]c∈[b]. We have shown that any member of [a][a][a] is a member of [b][b][b] and therefore that [a]⊆[b][a]\subseteq[b][a]⊆[b]. The fact that [b]⊆[a][b]\subseteq[a][b]⊆[a] can be shown in the same way. We deduce that [a]=[b][a]=[b][a]=[b], which proves the theorem.

The point of this theorem is that if we can find a binary relation that satisfies certain properties, namely the properties of an equivalence relation, then we can classify things into categories, where the categories are the equivalence classes.

For example, suppose that UUU is a possibly infinite set. Define a binary relation ∼\sim∼ on P(U){\mathscr P}(U)P(U) as follows: For XXX and YYY in P(U){\mathscr P}(U)P(U), X∼YX\sim YX∼Y if and only if there is a bijective function from the set XXX to the set YYY. In other words, X∼YX\sim YX∼Y means that XXX and YYY have the same cardinality. Then ∼\sim∼ is an equivalence relation on P(U){\mathscr P}(U)P(U). (The symbol ∼\sim∼ is often used to denote equivalence relations. It is usually read "is equivalent to.") If X∈P(U)X\in{\mathscr P}(U)X∈P(U), then the equivalence class [X]∼[X]_{\sim}[X]∼​ consists of all the subsets of UUU that have the same cardinality as XXX. We have classified all the subsets of UUU according to their cardinality—even though we have never said what an infinite cardinality is. (We have only said what it means to have the same cardinality.)

You might remember a popular puzzle called Rubik's Cube, a cube made of smaller cubes with colored sides that could be manipulated by twisting layers of little cubes. The object was to manipulate the cube so that the colors of the little cubes formed a certain configuration. Define two configurations of the cube to be equivalent if it's possible to manipulate one configuration into the other by a sequence of twists. This is, in fact, an equivalence relation on the set of possible configurations. (Symmetry follows from the fact that each move is reversible.) It has been shown that this equivalence relation has exactly twelve equivalence classes. The interesting fact is that it has more than one equivalence class: If the configuration that the cube is in and the configuration that you want to achieve are not in the same equivalence class, then you are doomed to failure.

Transitive Closure#

Suppose that R\mathscr RR is a binary relation on a set AAA. Even though R\mathscr RR might not be transitive, it is always possible to construct a transitive relation from R\mathscr RR in a natural way. If we think of a R ba\,{\mathscr R}\,baRb as meaning that aaa is related by R\mathscr RR to bbb "in one step," then we consider the relationship that holds between two elements xxx and yyy when xxx is related by R\mathscr RR to yyy "in one or more steps." This relationship defines a binary relation on AAA that is called the transitive closure of R\mathscr RR. The transitive closure of R\mathscr RR is denoted R∗{\mathscr R}^*R∗. Formally, R∗{\mathscr R}^*R∗ is defined as follows: For aaa and bbb in AAA, a R∗ ba\,{\mathscr R}^*\,baR∗b if there is a sequence x0,x1,…xnx_0,x_1,\dots x_nx0​,x1​,…xn​ of elements of AAA, where n>0n>0n>0 and x0=ax_0=ax0​=a and xn=bx_n=bxn​=b, such that x0 R x1x_0\,{\mathscr R}\,x_1x0​Rx1​, x1 R x2x_1\,{\mathscr R}\,x_2x1​Rx2​, \dots, and xn−1 R xnx_{n-1}\,{\mathscr R}\,x_nxn−1​Rxn​.

For example, if a R ca\,{\mathscr R}\,caRc, c R dc\,{\mathscr R}\,dcRd, and d R bd\,{\mathscr R}\,bdRb, then we would have that a R∗ ba\,{\mathscr R}^*\,baR∗b. Of course, we would also have that a R∗ ca\,{\mathscr R}^*\,caR∗c, and a R∗ da\,{\mathscr R}^*\,daR∗d.

For a practical example, suppose that CCC is the set of all cities and let A\mathscr AA be the binary relation on CCC such that for xxx and yyy in CCC, x A yx\,{\mathscr A}\,yxAy if there is a regularly scheduled airline flight from xxx to yyy. Then the transitive closure A∗{\mathscr A}^*A∗ has a natural interpretation: x A∗ yx\,{\mathscr A}^*\,yxA∗y if it's possible to get from xxx to yyy by a sequence of one or more regularly scheduled airline flights. You'll find a few more examples of transitive closures in the exercises.

  3. Consider each of the following relations on the set of people. Is the relation reflexive? Symmetric? Transitive? Is it an equivalence relation?

    • xxx is related to yyy if xxx and yyy have the same biological parents.
      Answer

      This is reflexive, symmetric, transitive, and hence an equivalence relation.

    • xxx is related to yyy if xxx and yyy have at least one biological parent in common.
      Answer

      This is reflexive and symmetric, but not transitive (Alice and Bob might share a mother, while Bob and Carla share a father, while Alice and Carla have no parent in common).

    • xxx is related to yyy if xxx and yyy were born in the same year.
      Answer

      This is reflexive, symmetric, transitive, and hence an equivalence relation.

    • xxx is related to yyy if xxx is taller than yyy.

      This is transitive only.

    • xxx is related to yyy if xxx and yyy have both visited Honolulu.
      Answer

      This is reflexive, symmetric, transitive, and hence an equivalence relation.

  4. It is possible for a relation to be both symmetric and antisymmetric. For example, the equality relation, ===, is a relation on any set which is both symmetric and antisymmetric. Suppose that AAA is a set and R\mathscr RR is a relation on AAA that is both symmetric and antisymmetric. Show that R\mathscr RR is a subset of === (when both relations are considered as sets of ordered pairs). That is, show that for any aaa and bbb in AAA, (a R b)→(a=b)(a\,{\mathscr R}\,b)\rightarrow (a=b)(aRb)→(a=b).

    Answer

    If we have a R ba\,{\mathscr R}\,baRb, then we also know b R ab\,{\mathscr R}\,abRa, because R\mathscr RR is symmetric. However, that can only be true if a=ba=ba=b, because R\mathscr RR is antisymmetric.
