Predicates and Quantifiers

(Content adapted from Critchlow & Eck)

In propositional logic, we can let ppp stand for "Roses are red" and qqq stand for "Violets are blue." Then p∧qp\land qp∧q will stand for "Roses are red and violets are blue." But we lose a lot in the translation into logic. Since propositional logic only deals with truth values, there's nothing we can do with ppp and qqq in propositional logic that has anything to do with roses, violets, or color. To apply logic to such things, we need predicates. The type of logic that uses predicates is called predicate logic, or, when the emphasis is on manipulating and reasoning with predicates, predicate calculus.

A predicate is a kind of incomplete proposition, which becomes a proposition when it is applied to some entity (or, as we'll see later, to several entities). In the proposition "the rose is red," the predicate is is red. By itself, "is red" is not a proposition. Think of it as having an empty slot, that needs to be filled in to make a proposition: "— is red." In the proposition "the rose is red," the slot is filled by the entity "the rose," but it could just as well be filled by other entities: "the barn is red"; "the wine is red"; "the banana is red." Each of these propositions uses the same predicate, but they are different propositions and they can have different truth values.

If PPP is a predicate and aaa is an entity, then P(a)P(a)P(a) stands for the proposition that is formed when PPP is applied to aaa. If PPP represents "is red" and aaa stands for "the rose," then P(a)P(a)P(a) is "the rose is red." If MMM is the predicate "is mortal" and sss is "Socrates," then M(s)M(s)M(s) is the proposition "Socrates is mortal."

Now, you might be asking, just what is an entity anyway? I am using the term here to mean some specific, identifiable thing to which a predicate can be applied. Generally, it doesn't make sense to apply a given predicate to every possible entity, but only to entities in a certain category. For example, it probably doesn't make sense to apply the predicate "is mortal" to your living room sofa. This predicate only applies to entities in the category of living things, since there is no way something can be mortal unless it is alive. This category is called the domain of discourse for the predicate.1

We are now ready for a formal definition of one-place predicates. A one-place predicate, like all the examples we have seen so far, has a single slot which can be filled in with one entity:

A one-place predicate associates a proposition with each entity in some collection of entities. This collection is called the domain of discourse for the predicate. If PPP is a predicate and aaa is an entity in the domain of discourse for PPP, then P(a)P(a)P(a) denotes the proposition that is associated with aaa by PPP. We say that P(a)P(a)P(a) is the result of applying PPP to aaa.

We can obviously extend this to predicates that can be applied to two or more entities. In the proposition "John loves Mary," loves is a two-place predicate. Besides John and Mary, it could be applied to other pairs of entities: "John loves Jane," "Bill loves Mary," "John loves Bill," "John loves John." If QQQ is a two-place predicate, then Q(a,b)Q(a,b)Q(a,b) denotes the proposition that is obtained when QQQ is applied to the entities aaa and bbb. Note that each of the "slots" in a two-place predicate can have its own domain of discourse. For example, if QQQ represents the predicate "owns," then Q(a,b)Q(a,b)Q(a,b) will only make sense when aaa is a person and bbb is an inanimate object. An example of a three-place predicate is "aaa gave bbb to ccc," and a four-place predicate would be "aaa bought bbb from ccc for ddd dollars." But keep in mind that not every predicate has to correspond to an English sentence.

When predicates are applied to entities, the results are propositions, and all the operators of propositional logic can be applied to these propositions just as they can to any propositions. Let RRR be the predicate "is red," and let LLL be the two-place predicate "loves." If aaa, bbb, jjj, mmm, and bbb are entities belonging to the appropriate categories, then we can form compound propositions such as:

#Quantifiers

Let's go back to the proposition with which we started this section: "Roses are red." This sentence is more difficult to handle than it might appear. We still can't express it properly in logic. The problem is that this proposition is not saying something about some particular entity. It really says that all roses are red (which happens to be a false statement, but that's what it means). Predicates can only be applied to individual entities.

Many other sentences raise similar difficulties: "All persons are mortal." "Some roses are red, but no roses are black." "All math courses are interesting." "Every prime number greater than two is odd." Words like all, no, some, and every are called quantifiers. We need to be able to express similar concepts in logic.

Suppose that PPP is a predicate, and we want to express the proposition that PPP is true when applied to any entity in the domain of discourse. That is, we want to say "for any entity xxx in the domain of discourse, P(x)P(x)P(x) is true." In predicate logic, we write this in symbols as ∀x(P(x))\forall x(P(x))∀x(P(x)). The ∀\forall∀ symbol, which looks like an upside-down A, is usually read "for all," so that ∀x(P(x))\forall x(P(x))∀x(P(x)) is read as "for all xxx, P(x)P(x)P(x)." (It is understood that this means for all xxx in the domain of discourse for PPP.) For example, if RRR is the predicate "is red" and the domain of discourse consists of all roses, then ∀x(R(x))\forall x(R(x))∀x(R(x)) expresses the proposition "All roses are red." Note that the same proposition could be expressed in English as "Every rose is red" or "Any rose is red."

Now, suppose we want to say that a predicate, PPP, is true for some entity in its domain of discourse. This is expressed in predicate logic as ∃x(P(x))\exists x(P(x))∃x(P(x)). The ∃\exists∃ symbol, which looks like a backwards E, is usually read "there exists," but a more exact reading would be "there is at least one." Thus, ∃x(P(x))\exists x(P(x))∃x(P(x)) is read as "There exists an xxx such that P(x)P(x)P(x)," and it means "there is at least one xxx in the domain of discourse for PPP for which P(x)P(x)P(x) is true." If, once again, RRR stands for "is red" and the domain of discourse is "roses," then ∃x(R(x))\exists x(R(x))∃x(R(x)) could be expressed in English as "There is a red rose" or "At least one rose is red" or "Some rose is red." It might also be expressed as "Some roses are red," but the plural is a bit misleading since ∃x(R(x))\exists x(R(x))∃x(R(x)) is true even if there is only one red rose. We can now give the formal definitions:

Suppose that PPP is a one-place predicate. Then ∀x(P(x))\forall x(P(x))∀x(P(x)) is a proposition, which is true if and only if P(a)P(a)P(a) is true for every entity aaa in the domain of discourse for PPP. And ∃x(P(x))\exists x(P(x))∃x(P(x)) is a proposition which is true if and only if there is at least one entity, aaa, in the domain of discourse for PPP for which P(a)P(a)P(a) is true. The ∀\forall∀ symbol is called the universal quantifier, and ∃\exists∃ is called the existential quantifier.

The xxx in ∀x(P(x))\forall x(P(x))∀x(P(x)) and ∃x(P(x))\exists x(P(x))∃x(P(x)) is a variable. (More precisely, it is an entity variable, since its value can only be an entity.) Note that a plain P(x)P(x)P(x)—without the ∀x\forall x∀x or ∃x\exists x∃x—is not a proposition. P(x)P(x)P(x) is neither true nor false because xxx is not some particular entity, but just a placeholder in a slot that can be filled in with an entity. P(x)P(x)P(x) would stand for something like the statement "xxx is red," which is not really a statement in English at all. But it becomes a statement when the xxx is replaced by some particular entity, such as "the rose." Similarly, P(x)P(x)P(x) becomes a proposition if some entity aaa is substituted for the xxx, giving P(a)P(a)P(a).2

An open statement is an expression that contains one or more entity variables, which becomes a proposition when entities are substituted for the variables. (An open statement has open "slots" that need to be filled in.) P(x)P(x)P(x) and "xxx is red" are examples of open statements that contain one variable. If LLL is a two-place predicate and xxx and yyy are variables, then L(x,y)L(x,y)L(x,y) is an open statement containing two variables. An example in English would be "xxx loves yyy." The variables in an open statement are called free variables. An open statement that contains xxx as a free variable can be quantified with ∀x\forall x∀x or ∃x\exists x∃x. The variable xxx is then said to be bound. For example, xxx is free in P(x)P(x)P(x) and is bound in ∀x(P(x))\forall x(P(x))∀x(P(x)) and ∃x(P(x))\exists x(P(x))∃x(P(x)). The free variable yyy in L(x,y)L(x,y)L(x,y) becomes bound in ∀y(L(x,y))\forall y(L(x,y))∀y(L(x,y)) and in ∃y(L(x,y))\exists y(L(x,y))∃y(L(x,y)).

Note that ∀y(L(x,y))\forall y(L(x,y))∀y(L(x,y)) is still an open statement, since it contains xxx as a free variable. Therefore, it is possible to apply the quantifier ∀x\forall x∀x or ∃x\exists x∃x to ∀y(L(x,y))\forall y(L(x,y))∀y(L(x,y)), giving ∀x(∀y(L(x,y)))\forall x\big(\forall y(L(x,y))\big)∀x(∀y(L(x,y))) and ∃x(∀y(L(x,y)))\exists x\big(\forall y(L(x,y))\big)∃x(∀y(L(x,y))). Since all the variables are bound in these expressions, they are propositions. If L(x,y)L(x,y)L(x,y) represents "xxx loves yyy," then ∀y(L(x,y))\forall y(L(x,y))∀y(L(x,y)) is something like "xxx loves everyone," and ∃x(∀y(L(x,y)))\exists x\big(\forall y(L(x,y))\big)∃x(∀y(L(x,y))) is the proposition, "There is someone who loves everyone." Of course, we could also have started with ∃x(L(x,y))\exists x(L(x,y))∃x(L(x,y)): "There is someone who loves yyy." Applying ∀y\forall y∀y to this gives ∀y(∃x(L(x,y)))\forall y\big(\exists x(L(x,y))\big)∀y(∃x(L(x,y))), which means "For every person, there is someone who loves that person." Note in particular that ∃x(∀y(L(x,y)))\exists x\big(\forall y(L(x,y))\big)∃x(∀y(L(x,y))) and ∀y(∃x(L(x,y)))\forall y\big(\exists x(L(x,y))\big)∀y(∃x(L(x,y))) do not mean the same thing. Altogether, there are eight different propositions that can be obtained from L(x,y)L(x,y)L(x,y) by applying quantifiers, with six distinct meanings among them.

(From now on, I will leave out parentheses when there is no ambiguity. For example, I will write ∀x P(x)\forall x\, P(x)∀xP(x) instead of ∀x(P(x))\forall x(P(x))∀x(P(x)) and ∃x ∃y L(x,y)\exists x\,\exists y\,L(x,y)∃x∃yL(x,y) instead of ∃x(∃y(L(x,y)))\exists x\big(\exists y(L(x,y))\big)∃x(∃y(L(x,y))). Furthermore, I will sometimes give predicates and entities names that are complete words instead of just letters, as in Red(x)Red(x)Red(x) and Loves(john,mary)Loves(john,mary)Loves(john,mary). This might help to make examples more readable.)

#Translating between Predicate Logic and English

In predicate logic, the operators and laws of Boolean algebra still apply. For example, if PPP and QQQ are one-place predicates and aaa is an entity in the domain of discourse, then P(a)→Q(a)P(a)\rightarrow Q(a)P(a)→Q(a) is a proposition, and it is logically equivalent to ¬P(a)∨Q(a)\lnot P(a)\lor Q(a)¬P(a)∨Q(a). Furthermore, if xxx is a variable, then P(x)→Q(x)P(x)\rightarrow Q(x)P(x)→Q(x) is an open statement, and ∀x(P(x)→Q(x))\forall x(P(x)\rightarrow Q(x))∀x(P(x)→Q(x)) is a proposition. So are P(a)∧(∃x Q(x))P(a)\land(\exists x\,Q(x))P(a)∧(∃xQ(x)) and (∀x P(x))→(∃xP(x))(\forall x\,P(x))\rightarrow(\exists xP(x))(∀xP(x))→(∃xP(x)). Obviously, predicate logic can be very expressive. Unfortunately, the translation between predicate logic and English sentences is not always obvious.

Let's look one more time at the proposition "Roses are red." If the domain of discourse consists of roses, this translates into predicate logic as ∀x Red(x)\forall x\, Red(x)∀xRed(x). However, the sentence makes more sense if the domain of discourse is larger—for example if it consists of all flowers. Then "Roses are red" has to be read as "All flowers which are roses are red," or "For any flower, if that flower is a rose, then it is red." The last form translates directly into logic as ∀x(Rose(x)→Red(x))\forall x\big(Rose(x)\rightarrow Red(x)\big)∀x(Rose(x)→Red(x)). Suppose we want to say that all red roses are pretty. The phrase "red rose" is saying both that the flower is a rose and that it is red, and it must be translated as a conjunction, Rose(x)∧Red(x)Rose(x)\land Red(x)Rose(x)∧Red(x). So, "All red roses are pretty" can be rendered as ∀x((Rose(x)∧Red(x))→Pretty(x))\forall x\big((Rose(x)\land Red(x))\rightarrow Pretty(x)\big)∀x((Rose(x)∧Red(x))→Pretty(x)).

Here are a few more examples of translations from predicate logic to English. Let H(x)H(x)H(x) represent "xxx is happy," let C(y)C(y)C(y) represent "yyy is a computer," and let O(x,y)O(x,y)O(x,y) represent "xxx owns yyy." (The domain of discourse for xxx consists of people, and the domain for yyy consists of inanimate objects.) Then we have the following translations:

  • Jack owns a computer: ∃x(O(jack,x)∧C(x))\exists x\big(O(jack,x)\land C(x)\big)∃x(O(jack,x)∧C(x)). (That is, there is at least one thing such that Jack owns that thing and that thing is a computer.)
  • Everything Jack owns is a computer: ∀x(O(jack,x)→C(x))\forall x\big(O(jack,x)\rightarrow C(x)\big)∀x(O(jack,x)→C(x)).
  • If Jack owns a computer, then he's happy: (∃y(O(jack,y)∧C(y)))→H(jack)\big(\exists y(O(jack,y)\land C(y))\big)\rightarrow H(jack)(∃y(O(jack,y)∧C(y)))→H(jack).
  • Everyone who owns a computer is happy: ∀x( (∃y(O(x,y)∧C(y))→H(x)) )\forall x\big(\,\big(\exists y(O(x,y)\land C(y)\big)\rightarrow H(x)\big)\,\big)∀x((∃y(O(x,y)∧C(y))→H(x))).
  • Everyone owns a computer: ∀x ∃y(C(y)∧O(x,y))\forall x\,\exists y\big(C(y)\land O(x,y)\big)∀x∃y(C(y)∧O(x,y)). (Note that this allows each person to own a different computer. The proposition ∃y ∀x(C(y)∧O(x,y))\exists y\,\forall x\big(C(y)\land O(x,y)\big)∃y∀x(C(y)∧O(x,y)) would mean that there is a single computer which is owned by everyone.)
  • Everyone is happy: ∀xH(x)\forall xH(x)∀xH(x).
  • Everyone is unhappy: ∀x(¬H(x))\forall x(\lnot H(x))∀x(¬H(x)).
  • Someone is unhappy: ∃x(¬H(x))\exists x(\lnot H(x))∃x(¬H(x)).
  • At least two people are happy: ∃x∃y(H(x)∧H(y)∧(x≠y))\exists x \exists y\big(H(x) \land H(y) \land (x\ne y)\big)∃x∃y(H(x)∧H(y)∧(x​=y)). (The stipulation that x≠yx\ne yx​=y is necessary because two different variables can refer to the same entity. The proposition ∃x∃y(H(x)∧H(y))\exists x\exists y(H(x)\land H(y))∃x∃y(H(x)∧H(y)) is true even if there is only one happy person.)
  • There is exactly one happy person: (∃xH(x)))∧(∀y∀z((H(y)∧H(z))→(y=z)))\big(\exists x H(x)\big)) \land \big(\forall y \forall z((H(y)\land H(z))\rightarrow (y=z))\big)(∃xH(x)))∧(∀y∀z((H(y)∧H(z))→(y=z))). (The first part of this conjunction says that there is at least one happy person. The second part says that if yyy and zzz are both happy people, then they are actually the same person. That is, it's not possible to find two \emph{different} people who are happy.)

#Logical Equivalence

To calculate in predicate logic, we need a notion of logical equivalence. Clearly, there are pairs of propositions in predicate logic that mean the same thing. Consider the propositions ¬(∀xH(x))\lnot(\forall x H(x))¬(∀xH(x)) and ∃x(¬H(x))\exists x(\lnot H(x))∃x(¬H(x)), where H(x)H(x)H(x) represents "xxx is happy." The first of these propositions means "Not everyone is happy," and the second means "Someone is not happy." These statements have the same truth value: If not everyone is happy, then someone is unhappy and vice versa. But logical equivalence is much stronger than just having the same truth value. In propositional logic, logical equivalence is defined in terms of propositional variables: two compound propositions are logically equivalent if they have the same truth values for all possible truth values of the propositional variables they contain. In predicate logic, two formulas are logically equivalent if they have the same truth value for all possible predicates.

Consider ¬(∀xP(x))\lnot(\forall x P(x))¬(∀xP(x)) and ∃x(¬P(x))\exists x(\lnot P(x))∃x(¬P(x)). These formulas make sense for any predicate PPP, and for any predicate PPP they have the same truth value. Unfortunately, we can't—as we did in propositional logic—just check this fact with a truth table: there are no subpropositions, connected by ∧\land∧, ∨\lor∨, etc, out of which to build a table. So, let's reason it out: To say ¬(∀xP(x))\lnot(\forall x P(x))¬(∀xP(x)) is true is just to say that it is not the case that P(x)P(x)P(x) is true for all possible entities xxx. So, there must be some entity aaa for which P(a)P(a)P(a) is false. Since P(a)P(a)P(a) is false, ¬P(a)\lnot P(a)¬P(a) is true. But saying that there is an aaa for which ¬P(a)\lnot P(a)¬P(a) is true is just saying that ∃x(¬P(x))\exists x(\lnot P(x))∃x(¬P(x)) is true. So, the truth of ¬(∀xP(x))\lnot(\forall x P(x))¬(∀xP(x)) implies the truth of ∃x(¬P(x))\exists x (\lnot P(x))∃x(¬P(x)). On the other hand, if ¬(∀xP(x))\lnot(\forall x P(x))¬(∀xP(x)) is false, then ∀xP(x)\forall x P(x)∀xP(x) is true. Since P(x)P(x)P(x) is true for every xxx, ¬P(x)\lnot P(x)¬P(x) is false for every xxx; that is, there is no entity aaa for which the statement ¬P(a)\lnot P(a)¬P(a) is true. But this just means that the statement ∃x(¬P(x))\exists x(\lnot P(x))∃x(¬P(x)) is false. In any case, then, the truth values of ¬(∀xP(x))\lnot(\forall x P(x))¬(∀xP(x)) and ∃x(¬P(x))\exists x(\lnot P(x))∃x(¬P(x)) are the same. Since this is true for any predicate PPP, we will say that these two formulas are logically equivalent and write ¬(∀xP(x))≡∃x(¬P(x))\lnot(\forall x P(x)) \equiv \exists x(\lnot P(x))¬(∀xP(x))≡∃x(¬P(x)).

A similar argument would show that ¬(∃xP(x))≡∀x(¬P(x))\lnot(\exists x P(x)) \equiv \forall x(\lnot P(x))¬(∃xP(x))≡∀x(¬P(x)). These two equivalences, which explicate the relation between negation and quantification, are known as De Morgan's Laws for predicate logic. (They are closely related to De Morgan's Laws for propositional logic; see the exercises.) These laws can be used to help simplify expressions. For example,

¬ ∀y(R(y)∨Q(y))\lnot\,\forall y (R(y)\lor Q(y))¬∀y(R(y)∨Q(y))≡∃y(¬(R(y)∨Q(y)))\equiv \exists y(\lnot(R(y)\lor Q(y)))≡∃y(¬(R(y)∨Q(y)))
≡∃y((¬R(y))∧(¬Q(y))\equiv \exists y((\lnot R(y))\land(\lnot Q(y))≡∃y((¬R(y))∧(¬Q(y))

It might not be clear exactly why this qualifies as a "simplification," but it's generally considered simpler to have the negation operator applied to basic propositions such as R(y)R(y)R(y), rather than to quantified expressions such as ∀y(R(y)∨Q(y))\forall y (R(y)\lor Q(y))∀y(R(y)∨Q(y)). For a more complicated example:

¬ ∃x(P(x)∧(∀y(Q(y)→Q(x))))\lnot\,\exists x\big(P(x)\land (\forall y (Q(y)\rightarrow Q(x)))\big)¬∃x(P(x)∧(∀y(Q(y)→Q(x))))≡∀x(¬(P(x)∧(∀y(Q(y)→Q(x))))\equiv\forall x\big(\lnot\big(P(x)\land (\forall y (Q(y)\rightarrow Q(x)))\big)≡∀x(¬(P(x)∧(∀y(Q(y)→Q(x))))

PPP can be any one-place predicate, and QQQ can be any two-place predicate.

De Morgan

¬ (∀xP(x))≡∃x(¬P(x))\lnot\,(\forall x P(x)) \equiv \exists x(\lnot P(x))¬(∀xP(x))≡∃x(¬P(x))

¬ (∃xP(x))≡∀x(¬P(x))\lnot\,(\exists x P(x)) \equiv \forall x(\lnot P(x))¬(∃xP(x))≡∀x(¬P(x))

Interchange

∀x∀yQ(x,y)≡∀y∀xQ(x,y)\forall x \forall y Q(x,y) \equiv \forall y \forall x Q(x,y)∀x∀yQ(x,y)≡∀y∀xQ(x,y)

∃x∃yQ(x,y)≡∃y∃xQ(x,y)\exists x \exists y Q(x,y) \equiv \exists y \exists x Q(x,y)∃x∃yQ(x,y)≡∃y∃xQ(x,y)

To define logical equivalence in predicate logic more formally, we need to talk about formulas that contain predicate variables, that is, variables that act as place-holders for arbitrary predicates in the same way that propositional variables are place-holders for propositions and entity variables are place-holders for entities. With this in mind, we can define logical equivalence and the closely related concept of tautology for predicate logic.

Let P\mathscr{P}P be a formula of predicate logic which contains one or more predicate variables. P\mathscr{P}P is said to be a tautology if it is true whenever all the predicate variables that it contains are replaced by actual predicates. Two formulas P\mathscr{P}P and Q\mathscr{Q}Q are said to be logically equivalent if P↔Q\mathscr{P}\leftrightarrow\mathscr{Q}P↔Q is a tautology, that is if P\mathscr{P}P and Q\mathscr{Q}Q always have the same truth value when the predicate variables they contain are replaced by actual predicates. The notation P≡Q\mathscr{P}\equiv\mathscr{Q}P≡Q asserts that P\mathscr{P}P is logically equivalent to Q\mathscr{Q}Q.
