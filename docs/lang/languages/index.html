Languages

(Content adapted from Critchlow & Eck)

With the set of mathematical tools from the first two chapters, we are now ready to study languages and formal language theory. Our intent is to examine the question of how, and which, languages can be mechanically generated and recognized; and, ultimately, to see what this tells us about what computers can and can't do.

In formal language theory, an alphabet is a finite, non-empty set. The elements of the set are called symbols. A finite sequence of symbols a1a2…ana_1a_2\ldots a_na1​a2​…an​ from an alphabet is called a string over that alphabet.

For example, Σ={0,1}\Sigma = \{0,1\}Σ={0,1} is an alphabet, and 011, 1010, and 1 are all strings over Σ\SigmaΣ.

Note that strings really are sequences of symbols, which implies that order matters. Thus 011, 101, and 110 are all different strings, though they are made up of the same symbols. The strings x=a1a2…amx=a_1a_2\ldots a_mx=a1​a2​…am​ and y=b1b2…bny=b_1b_2\ldots b_ny=b1​b2​…bn​ are equal only if m=nm=nm=n (i.e., the strings contain the same number of symbols) and ai=bia_i=b_iai​=bi​ for all 1≤i≤n1 \leq i \leq n1≤i≤n.

#String Operations

Just as there are operations defined on numbers, truth values, sets, and other mathematical entities, there are operations defined on strings. Some important operations are:

  • length: the length of a string xxx is the number of symbols in it. The notation for the length of xxx is ∣x∣|x|∣x∣. Note that this is consistent with other uses of ∣ ∣|\ |∣ ∣, all of which involve some notion of size: ∣number∣|number|∣number∣ measures how big a number is (in terms of its distance from 0); ∣set∣|set|∣set∣ measures the size of a set (in terms of the number of elements).

    We will occasionally refer to a length-n string. This is a slightly awkward, but concise, shorthand for "a string whose length is nnn".

  • concatenation: the concatenation of two strings x=a1a2…amx=a_1 a_2\ldots a_mx=a1​a2​…am​ and y=b1b2…bny=b_1b_2\ldots b_ny=b1​b2​…bn​ is the sequence of symbols a1…amb1…bna_1\ldots a_mb_1\ldots b_na1​…am​b1​…bn​. Sometimes ⋅\cdot⋅ is used to denote concatenation, but it is far more usual to see the concatenation of xxx and yyy denoted by xyxyxy than by x⋅yx\cdot yx⋅y. You can easily convince yourself that concatenation is associative (i.e., (xy)z=x(yz)(xy)z = x(yz)(xy)z=x(yz) for all strings x,yx,yx,y and zzz). Concatenation is not commutative (i.e., it is not always true that xy=yxxy = yxxy=yx: for example, if x=ax=ax=a and y=by=by=b then xy=abxy=abxy=ab while yx=bayx=bayx=ba and, as discussed above, these strings are not equal).

  • reversal: the reverse of a string x=a1a2…anx=a_1a_2\ldots a_nx=a1​a2​…an​ is the string xR=anan−1…a2a1x^R = a_na_{n-1}\ldots a_2a_1xR=an​an−1​…a2​a1​.

For example, let Σ={a,b}\Sigma = \{a,b\}Σ={a,b}, x=ax=ax=a, y=abaay=abaay=abaa, and z=babz=babz=bab. Then ∣x∣=1|x| = 1∣x∣=1, ∣y∣=4|y| = 4∣y∣=4, and ∣z∣=3|z|=3∣z∣=3. Also, xx=aaxx = aaxx=aa, xy=aabaaxy = aabaaxy=aabaa, xz=ababxz = ababxz=abab, and zx=babazx = babazx=baba. Finally, xR=ax^R = axR=a, yR=aabay^R = aabayR=aaba, and zR=babz^R=babzR=bab.

By the way, the previous example illustrates a naming convention standard throughout language theory texts: if a letter is intended to represent a single symbol in an alphabet, the convention is to use a letter from the beginning of the English alphabet (aaa, bbb, ccc, ddd); if a letter is intended to represent a string, the convention is to use a letter from the end of the English alphabet (uuu, vvv, etc.).

#Empty String

In set theory, we have a special symbol to designate the set that contains no elements. Similarly, language theory has a special symbol ε\varepsilonε which is used to represent the empty string, the string with no symbols in it. (Some texts use the symbol λ\lambdaλ instead.) It is worth noting that ∣ε∣=0|\varepsilon| = 0∣ε∣=0, that εR=ε\varepsilon^R = \varepsilonεR=ε, and that ε⋅x=x⋅ε=x\varepsilon \cdot x = x \cdot \varepsilon = xε⋅x=x⋅ε=x for all strings xxx. (This last fact may appear a bit confusing. Remember that ε\varepsilonε is not a symbol in a string with length 1, but rather the name given to the string made up of 0 symbols. Pasting those 0 symbols onto the front or back of a string xxx still produces xxx.)

The set of all strings over an alphabet Σ\SigmaΣ is denoted Σ∗\Sigma^*Σ∗. (In language theory, the symbol ∗^*∗ is typically used to denote "zero or more", so Σ∗\Sigma^*Σ∗ is the set of strings made up of zero or more symbols from Σ\SigmaΣ.) Note that while an alphabet Σ\SigmaΣ is by definition a finite set of symbols, and strings are by definition finite sequences of those symbols, the set Σ∗\Sigma^*Σ∗ is always infinite (as long as Σ\SigmaΣ is not empty). Why is this? Suppose Σ\SigmaΣ contains nnn elements. Then there is one string over Σ\SigmaΣ with 0 symbols, nnn strings with 1 symbol, n2n^2n2 strings with 2 symbols (since there are nnn choices for the first symbol and nnn choices for the second), n3n^3n3 strings with 3 symbols, etc.

For example, if Σ={1}\Sigma = \{1\}Σ={1}, then Σ∗={ε,1,11,111,…}\Sigma^* = \{\varepsilon, 1, 11, 111, \ldots\}Σ∗={ε,1,11,111,…}. If Σ={a,b}\Sigma = \{a,b\}Σ={a,b}, then Σ∗={ε,a,b,aa,ab,ba,bb,aaa,aab,…}\Sigma^* = \{ \varepsilon, a, b, aa, ab, ba, bb, aaa, aab, \ldots\}Σ∗={ε,a,b,aa,ab,ba,bb,aaa,aab,…}.

We now come to the definition of a language in the formal language theoretical sense.

A language over an alphabet Σ\SigmaΣ is a subset of Σ∗\Sigma^*Σ∗. Thus, a language over Σ\SigmaΣ is an element of P(Σ∗){\mathscr P}(\Sigma^*)P(Σ∗), the power set of Σ∗\Sigma^*Σ∗.

In other words, any set of strings (over alphabet Σ\SigmaΣ) constitutes a language (over alphabet Σ\SigmaΣ).

For example, let Σ={0,1}\Sigma = \{0,1\}Σ={0,1}. Then the following are all languages over Σ\SigmaΣ:

  • L1={011,1010,111}L_1 = \{011, 1010, 111\}L1​={011,1010,111}

  • L2={0,10,110,1110,11110,…}L_2 = \{0, 10, 110, 1110, 11110, \ldots\}L2​={0,10,110,1110,11110,…}

  • L3={x∈Σ∗ ∣ n0(x)=n1(x)}L_3 = \{x \in \Sigma^* \ | \ n_0(x) = n_1(x) \}L3​={x∈Σ∗ ∣ n0​(x)=n1​(x)}, where the notation n0(x)n_0(x)n0​(x) stands for the number of 0's in the string xxx, and similarly for n1(x)n_1(x)n1​(x).

  • L4={x ∣x represents a multiple of 5 in binary}L_4 = \{x \ | x\textrm{ represents a multiple of 5 in binary}\}L4​={x ∣x represents a multiple of 5 in binary}

Note that languages can be either finite or infinite. Because Σ∗\Sigma^*Σ∗ is infinite, it clearly has an infinite number of subsets, and so there are an infinite number of languages over Σ\SigmaΣ.

A programming language, such as Java or ReasonML, may be viewed as a formal language: if Σ\SigmaΣ is the set of all ASCII (or, more generally, Unicode) characters, then the set of all valid programs in a particular programming language is a language over Σ\SigmaΣ. We will see later how to formally specify such languages, but one step is often to describe smaller languages, such as the set of all keywords in the language (for Java, it is the 51-element set {\{{abstract, assert, boolean, …, while, _}\}}), or the set of valid identifiers (in Java, a valid identifier starts with a letter, $, or _, and continues with zero or more letters, digits, $, or _1, except it may not be a keyword or reserved word such as true, false, or null).

Languages are sets and therefore, as for any sets, it makes sense to talk about the union, intersection, and complement of languages. (When taking the complement of a language over an alphabet Σ\SigmaΣ, we always consider the univeral set to be Σ∗\Sigma^*Σ∗, the set of all strings over Σ\SigmaΣ.) Because languages are sets of strings, there are additional operations that can be defined on languages, operations that would be meaningless on more general sets. For example, the idea of concatenation can be extended from strings to languages.

For two sets of strings SSS and TTT, we define the concatenation of SSS and TTT (denoted S⋅TS\cdot TS⋅T or just STSTST) to be the set ST={st ∣ s∈S∧t∈T}ST = \{ st \ | \ s \in S \land t \in T \}ST={st ∣ s∈S∧t∈T}. For example, if S={ab,aab}S = \{ab, aab\}S={ab,aab} and T={ε,110,1010}T=\{\varepsilon, 110, 1010\}T={ε,110,1010}, then ST={ab,ab110,ab1010,aab,aab110,aab1010}ST = \{ab, ab110, ab1010, aab, aab110, aab1010\}ST={ab,ab110,ab1010,aab,aab110,aab1010}.
Note in particular that ab∈STab \in STab∈ST, because ab∈Sab \in Sab∈S, ε∈T\varepsilon \in Tε∈T, and ab⋅ε=abab \cdot \varepsilon = abab⋅ε=ab. Because concatenation of sets is defined in terms of the concatenation of the strings that the sets contain, concatenation of sets is associative and not commutative. (This can easily be verified.)

When a set SSS is concatenated with itself, the notation SSSSSS is usually scrapped in favour of S2S^2S2; if S2S^2S2 is concatenated with SSS, we write S3S^3S3 for the resulting set, etc. So S2S^2S2 is the set of all strings formed by concatenating two (possibly different, possibly identical) strings from SSS, S3S^3S3 is the set of strings formed by concatenating three strings from SSS, etc. Extending this notation, we take S1S^1S1 to be the set of strings formed from one string in SSS (i.e., S1S^1S1 is SSS itself), and S0S^0S0 to be the set of strings formed from zero strings in SSS (i.e., S0={ε}S^0 = \{\varepsilon\}S0={ε}). If we take the union S0∪S1∪S2∪…S^0 \cup S^1 \cup S^2 \cup \ldotsS0∪S1∪S2∪…, then the resulting set is the set of all strings formed by concatenating zero or more strings from SSS, and is denoted S∗S^*S∗. The set S∗S^*S∗ is called the Kleene closure2 of SSS, and the ∗^*∗ operator is called the Kleene star operator.

For example, let S={01,ba}S = \{01, ba\}S={01,ba}. Then

  • S0={ε}S^0 = \{\varepsilon\}S0={ε}

  • S1={01,ba}S^1 = \{01, ba\}S1={01,ba}

  • S2={0101,01ba,ba01,baba}S^2 = \{0101, 01ba, ba01, baba\}S2={0101,01ba,ba01,baba}

  • S3={010101,0101ba,01ba01,01baba,ba0101,ba01ba,baba01,bababa}S^3 = \{010101, 0101ba, 01ba01, 01baba, ba0101, ba01ba, baba01, bababa\}S3={010101,0101ba,01ba01,01baba,ba0101,ba01ba,baba01,bababa}

etc., so S∗={ε,01,ba,0101,01ba,ba01,baba,010101,0101ba,…}.S^* =\{\varepsilon,01,ba,0101,01ba,ba01,baba,010101,0101ba,\ldots\}.S∗={ε,01,ba,0101,01ba,ba01,baba,010101,0101ba,…}.

Note that this is the second time we have seen the notation something∗\textit{something}^*something∗. We have previously seen that for an alphabet Σ\SigmaΣ, Σ∗\Sigma^*Σ∗ is defined to be the set of all strings over Σ\SigmaΣ. If you think of Σ\SigmaΣ as being a set of length-1 strings, and take its Kleene closure, the result is once again the set of all strings over Σ\SigmaΣ, and so the two notions of ∗^*∗ coincide.

For example, let Σ={a,b}\Sigma = \{a,b\}Σ={a,b}. Then

  • Σ0={ε}\Sigma^0 = \{\varepsilon\}Σ0={ε}

  • Σ1={a,b}\Sigma^1 = \{a,b\}Σ1={a,b}

  • Σ2={aa,ab,ba,bb}\Sigma^2 = \{aa, ab, ba, bb\}Σ2={aa,ab,ba,bb}

  • Σ3={aaa,aab,aba,abb,baa,bab,bba,bbb}\Sigma^3 = \{aaa, aab, aba, abb, baa, bab, bba, bbb\}Σ3={aaa,aab,aba,abb,baa,bab,bba,bbb}

etc., so Σ∗={ε,a,b,aa,ab,ba,bb,aaa,aab,aba,abb,baa,bab,…}.\Sigma^* =\{\varepsilon,a,b,aa,ab,ba,bb,aaa,aab,aba,abb,baa,bab,\ldots\}.Σ∗={ε,a,b,aa,ab,ba,bb,aaa,aab,aba,abb,baa,bab,…}.

    • S2S^2S2
    • S3S^3S3
    • S∗S^*S∗
    • STSTST
    • TSTSTS

  2. The reverse of a language LLL is defined to be LR={xR ∣ x∈L}L^R = \{ x^R \ | \ x \in L\}LR={xR ∣ x∈L}. Find SRS^RSR and TRT^RTR for the SSS and TTT in the preceding problem.
