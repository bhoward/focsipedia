Context-Free Grammars

(Content adapted from Critchlow & Eck)

Both natural languages, such as English, and the artificial languages used for programming have a structure known as grammar or syntax. In order to form legal sentences or programs, the parts of the language must be fit together according to certain rules. For natural languages, the rules are somewhat informal (although high-school English teachers might have us believe differently). For programming languages, the rules are absolute, and programs that violate the rules will be rejected by a compiler.

In this chapter, we will study formal grammars and languages defined by them. The languages we look at will, for the most part, be "toy" languages, compared to natural languages or even to programming languages, but the ideas and techniques are basic to any study of language. In fact, many of the ideas arose almost simultaneously in the 1950s in the work of linguists who were studying natural language and programmers who were looking for ways to specify the syntax of programming languages.

The grammars in this chapter are generative grammars. A generative grammar is a set of rules that can be used to generate all the legal strings in a language. We will also consider the closely related idea of parsing. To parse a string means to determine how that string can be generated according to the rules.

This chapter is a continuation of the preceding chapter. Like a regular expression, a grammar is a way to specify a possibly infinite language with a finite amount of information. In fact, we will see that every regular language can be specified by a certain simple type of grammar. We will also see that some languages that can be specified by grammars are not regular.

In its most general form, a grammar is a set of rewriting rules. A rewriting rule specifies that a certain string of symbols can be substituted for all or part of another string. If www and uuu are strings, then w⟶uw\longrightarrow uw⟶u is a rewriting rule that specifies that the string www can be replaced by the string uuu. The symbol "⟶\longrightarrow⟶" is read "can be rewritten as." Rewriting rules are also called production rules or productions, and "⟶\longrightarrow⟶" can also be read as "produces." For example, if we consider strings over the alphabet {a,b,c}\{a,b,c\}{a,b,c}, then the production rule aba⟶ccaba\longrightarrow ccaba⟶cc can be applied to the string abbabacabbabacabbabac to give the string abbcccabbcccabbccc. The substring abaabaaba in the string abbabacabbabacabbabac has been replaced with cccccc.

In a context-free grammar, every rewriting rule has the form A⟶wA\longrightarrow wA⟶w, where AAA is single symbol and www is a string of zero or more symbols. (The grammar is "context-free" in the sense that www can be substituted for AAA wherever AAA occurs in a string, regardless of the surrounding context in which AAA occurs.) The symbols that occur on the left-hand sides of production rules in a context-free grammar are called non-terminal symbols. By convention, the non-terminal symbols are usually uppercase letters. The strings on the right-hand sides of the production rules can include non-terminal symbols as well as other symbols, which are called terminal symbols. By convention, the terminal symbols are usually lowercase letters. Here are some typical production rules that might occur in context-free grammars:

A⟶aAbBS⟶SSC⟶AccB⟶bA⟶ε\begin{aligned} &A\longrightarrow aAbB\\ &S\longrightarrow SS\\ &C\longrightarrow Acc\\ &B\longrightarrow b\\ &A\longrightarrow\varepsilon \end{aligned}​A⟶aAbBS⟶SSC⟶AccB⟶bA⟶ε​

In the last rule in this list, ε\varepsilonε represents the empty string, as usual. For example, this rule could be applied to the string aBaAcAaBaAcAaBaAcA to produce the string aBacAaBacAaBacA. The first occurrence of the symbol AAA in aBaAcAaBaAcAaBaAcA has been replaced by the empty string—which is just another way of saying that the symbol has been dropped from the string.

In every context-free grammar, one of the non-terminal symbols is designated as the start symbol of the grammar. The start symbol is often, though not always, denoted by SSS. When the grammar is used to generate strings in a language, the idea is to start with a string consisting of nothing but the start symbol. Then a sequence of production rules is applied. Each application of a production rule to the string transforms the string to a new string. If and when this process produces a string that consists purely of terminal symbols, the process ends. That string of terminal symbols is one of the strings in the language generated by the grammar. In fact, the language consists precisely of all strings of terminal symbols that can be produced in this way.

As a simple example, consider a grammar that has three production rules: S⟶aSS\longrightarrow aSS⟶aS, S⟶bSS\longrightarrow bSS⟶bS, and S⟶bS\longrightarrow bS⟶b. In this example, SSS is the only non-terminal symbol, and the terminal symbols are aaa and bbb. Starting from the string SSS, we can apply any of the three rules of the grammar to produce either aSaSaS, bSbSbS, or bbb. Since the string bbb contains no non-terminals, we see that bbb is one of the strings in the language generated by this grammar. The strings aSaSaS and bSbSbS are not in that language, since they contain the non-terminal symbol SSS, but we can continue to apply production rules to these strings. From aSaSaS, for example, we can obtain aaSaaSaaS, abSabSabS, or ababab. From abSabSabS, we go on to obtain abaSabaSabaS, abbSabbSabbS, or abbabbabb. The strings ababab and abbabbabb are in the language generated by the grammar. It's not hard to see that any string of aaa's and bbb's that ends with a bbb can be generated by this grammar, and that these are the only strings that can be generated. That is, the language generated by this grammar is the regular language specified by the regular expression (a∣b)∗b(a | b)^{*}b(a∣b)∗b.

It's time to give some formal definitions of the concepts which we have been discussing.

A context-free grammar is a 4-tuple (V,Σ,P,S)(V,\Sigma,P,S)(V,Σ,P,S), where:

  1. VVV is a finite set of symbols. The elements of VVV are the non-terminal symbols of the grammar.

  2. Σ\SigmaΣ is a finite set of symbols such that V∩Σ=∅V\cap\Sigma=\emptysetV∩Σ=∅. The elements of Σ\SigmaΣ are the terminal symbols of the grammar.

  3. PPP is a set of production rules. Each rule is of the form A⟶wA\longrightarrow wA⟶w where AAA is one of the symbols in VVV and www is a string in the language (V∪Σ)∗(V\cup\Sigma)^*(V∪Σ)∗.

  4. S∈VS\in VS∈V. SSS is the start symbol of the grammar.

Even though this is the formal definition, grammars are often specified informally simply by listing the set of production rules. When this is done it is assumed, unless otherwise specified, that the non-terminal symbols are just the symbols that occur on the left-hand sides of production rules of the grammar. The terminal symbols are all the other symbols that occur on the right-hand sides of production rules. The start symbol is the symbol that occurs on the left-hand side of the first production rule in the list. Thus, the list of production rules

The following theorem states some simple facts about the relations ⟹\Longrightarrow⟹ and ⟹∗\Longrightarrow^*⟹∗:

We can use ⟹∗\Longrightarrow^*⟹∗ to give a formal definition of the language generated by a context-free grammar:

Suppose that G=(V,Σ,P,S)G=(V,\Sigma,P,S)G=(V,Σ,P,S) is a context-free grammar. Then the language generated by GGG is the language L(G)L(G)L(G) over the alphabet Σ\SigmaΣ defined by

L(G)={w∈Σ∗  ∣  S⟹G∗w}L(G)=\{w\in \Sigma^*\;|\; S\Longrightarrow_G^* w\}L(G)={w∈Σ∗∣S⟹G∗​w}

For another example, consider the language {anbm  ∣  n≥m≥0}\{a^nb^m\;|\; n\ge m\ge0\}{anbm∣n≥m≥0}. Let's try to design a grammar that generates this language. This is similar to the previous example, but now we want to include strings that contain more aaa's than bbb's. The production rule S⟶aSbS\longrightarrow aSbS⟶aSb always produces the same number of aaa's and bbb's. Can we modify this idea to produce more aaa's than bbb's?

One approach would be to produce a string containing just as many aaa's as bbb's, and then to add some extra aaa's. A rule that can generate any number of aaa's is A⟶aAA\longrightarrow aAA⟶aA. After applying the rule S⟶aSbS\longrightarrow aSbS⟶aSb for a while, we want to move to a new state in which we apply the rule A⟶aAA\longrightarrow aAA⟶aA. We can get to the new state by applying a rule S⟶AS\longrightarrow AS⟶A that changes the SSS into an AAA. We still need a way to finish the process, which means getting rid of all non-terminal symbols in the string. For this, we can use the rule A⟶εA\longrightarrow\varepsilonA⟶ε. Putting these rules together, we get the grammar

S⟶aSbS⟶AA⟶aAA⟶ε\begin{aligned} &S\longrightarrow aSb\\ &S\longrightarrow A\\ &A\longrightarrow aA\\ &A\longrightarrow \varepsilon \end{aligned}​S⟶aSbS⟶AA⟶aAA⟶ε​

This grammar does indeed generate the language {anbm  ∣  n≥m≥0}\{a^nb^m\;|\; n\ge m\ge 0\}{anbm∣n≥m≥0}. With slight variations on this grammar, we can produce other related languages. For example, if we replace the rule A⟶εA\longrightarrow \varepsilonA⟶ε with A⟶aA\longrightarrow aA⟶a, we get the language {anbm  ∣  n>m≥0}\{a^nb^m\;|\; n> m\ge 0\}{anbm∣n>m≥0}.

There are other ways to generate the language {anbm  ∣  n≥m≥0}\{a^nb^m\;|\; n\ge m\ge 0\}{anbm∣n≥m≥0}. For example, the extra non-terminal symbol, AAA, is not really necessary, if we allow SSS to sometimes produce a single aaa without a bbb. This leads to the grammar

S⟶aSbS⟶aSS⟶ε\begin{aligned} &S\longrightarrow aSb\\ &S\longrightarrow aS\\ &S\longrightarrow \varepsilon \end{aligned}​S⟶aSbS⟶aSS⟶ε​

The basic idea of this proof is that if www contains the same number of aaa's as bbb's, then an aaa at the beginning of www must have a "matching" bbb somewhere in www. This bbb matches the aaa in the sense that the corresponding rir_iri​ is zero, and the bbb marks the end of a string xxx which contains the same number of aaa's as bbb's. For example, in the string aababbabbaaababbabbaaababbabba, the aaa at the beginning of the string is matched by the third bbb, since aababbaababbaababb is the shortest prefix of aababbabbaaababbabbaaababbabba that has an equal number of aaa's and bbb's.

Closely related to this idea of matching aaa's and bbb's is the idea of balanced parentheses. Consider a string made up of parentheses, such as (()(()))(()). The parentheses in this sample string are balanced because each left parenthesis has a matching right parenthesis, and the matching pairs are properly nested. A careful definition uses the sort of integer sequence introduced in the above proof. Let www be a string of parentheses. Write w=c1c2⋯cnw=c_1c_2\cdots c_nw=c1​c2​⋯cn​, where each cic_ici​ is either ( or ). Define a sequence of integers r1r_1r1​, r2r_2r2​, \dots, rnr_nrn​, where rir_iri​ is the number of left parentheses in c1c2⋯cic_1c_2\cdots c_ic1​c2​⋯ci​ minus the number of right parentheses. The fact the ri≥0r_i\ge0ri​≥0 means that the nesting of pairs of parentheses is correct: You can't have a right parenthesis unless it is balanced by a left parenthesis in the preceding part of the string. Before leaving this section, we should look at a few more general results. Since we know that most operations on regular languages produce languages that are also regular, we can ask whether a similar result holds for context-free languages. We will see later that the intersection of two context-free languages is not necessarily context-free. Also, the complement of a context-free language is not necessarily context-free. However, some other operations on context-free languages do produce context-free languages.

Theorem: Suppose that LLL and MMM are context-free languages. Then the languages L∪ML\cup ML∪M, LMLMLM, and L∗L^*L∗ are also context-free.

Proof: I will prove only the first claim of the theorem, that L∪ML\cup ML∪M is context-free. In the exercises for this section, you are asked to construct grammars for LMLMLM and L∗L^*L∗ (without giving formal proofs that your answers are correct).

Once that decision is made, to make a string in LLL, we use production rules from GGG, while to make a string in MMM, we use rules from HHH. Finally, we should clarify the relationship between context-free languages and regular languages. We have already seen that there are context-free languages which are not regular. On the other hand, it turns out that every regular language is context-free. That is, given any regular language, there is a context-free grammar that generates that language. This means that any syntax that can be expressed by a regular expression, by a DFA, or by an NFA could also be expressed by a context-free grammar. In fact, we only need a certain restricted type of context-free grammar to duplicate the power of regular expressions.

A right-regular grammar is a context-free grammar in which the right-hand side of every production rule has one of the following forms: the empty string; a string consisting of a single non-terminal symbol; or a string consisting of a single terminal symbol followed by a single non-terminal symbol.

Examples of the types of production rule that are allowed in a right-regular grammar are A⟶εA\longrightarrow\varepsilonA⟶ε, B⟶CB\longrightarrow CB⟶C, and D⟶aED\longrightarrow aED⟶aE. The idea of the proof is that given a right-regular grammar, we can build a corresponding NFANFANFA and vice-versa. The states of the NFANFANFA correspond to the non-terminal symbols of the grammar. The start symbol of the grammar corresponds to the starting state of the NFA. A production rule of the form A⟶bCA\longrightarrow bCA⟶bC corresponds to a transition in the NFA from state AAA to state CCC while reading the symbol bbb. A production rule of the form A⟶BA\longrightarrow BA⟶B corresponds to an ε\varepsilonε-transition from state AAA to state BBB in the NFA. And a production rule of the form A⟶εA\longrightarrow\varepsilonA⟶ε exists in the grammar if and only if AAA is a final state in the NFA. With this correspondence, a derivation of a string www in the grammar corresponds to an execution path through the NFA as it accepts the string www. I won't give a complete proof here. You are welcome to work through the details if you want. But the important fact is:

Theorem: A language LLL is regular if and only if there is a right-regular grammar GGG such that L=L(G)L=L(G)L=L(G). In particular, every regular language is context-free.
