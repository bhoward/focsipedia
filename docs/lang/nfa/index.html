Nondeterministic Finite-State Automata

(Content adapted from Critchlow & Eck)

As mentioned briefly above, there is an alternative school of though as to what properties should be required of a finite-state automaton's transition function. Recall our motivating example of a C++ compiler and a legal if statement. In our description, we had the compiler in an "expecting a )" state; on seeing a ), the compiler moved into an "expecting a { or a legal statement" state. An alternative way to view this would be to say that the compiler, on seeing a ), could move into one of two different states: it could move to an "expecting a {" state or move to an "expecting a legal statement" state. Thus, from a single state, on input ), the compiler has multiple moves. This alternative interpretation is not allowed by the DFA model.

A second point on which one might question the DFA model is the fact that input must be consumed for the machine to change state. Think of the syntax for C++ function declarations. The return type of a function need not be specified (the default is taken to be int). The start state of the compiler when parsing a function declaration might be "expecting a return type"; then with no input consumed, the compiler can move to the state "expecting a legal function name". To model this, it might seem reasonable to allow transitions that do not require the consumption of input (such transitions are called ε\varepsilonε-transitions). Again, this is not supported by the DFA abstraction. There is, therefore, a second class of finite-state automata that people study, the class of nondeterministic finite-state automata.

A nondeterministic finite-state automaton (NFA) is the same as a deterministic finite-state automaton except that the transition function is no longer a function that maps a state-input pair to a state; rather, it maps a state-input pair or a state-ε\varepsilonε pair to a set of states. No longer do we have δ(q,a)=q′\delta(q,a) = q'δ(q,a)=q′, meaning that the machine must change to state q′q'q′ if it is in state qqq and consumes an aaa. Rather, we have ∂(q,a)={q1,q2,…,qn}\partial(q,a) = \{q_1, q_2, \ldots, q_n\}∂(q,a)={q1​,q2​,…,qn​}, meaning that if the machine is in state qqq and consumes an aaa, it might move directly to any one of the states q1,…,qnq_1, \ldots, q_nq1​,…,qn​. Note that the set of next states ∂(q,a)\partial(q,a)∂(q,a) is defined for every state qqq and every input symbol aaa, but for some qqq's and aaa's it could be empty, or contain just one state (there don't have to be multiple next states). The function ∂\partial∂ must also specify whether it is possible for the machine to make any moves without input being consumed, i.e., ∂(q,ε)\partial(q, \varepsilon)∂(q,ε) must be specified for every state qqq. Again, it is quite possible that ∂(q,ε)\partial(q, \varepsilon)∂(q,ε) may be empty for some states qqq: there need not be ε\varepsilonε-transitions out of qqq.

Formally, a nondeterministic finite-state automaton MMM is specified by 5 components: M=(Q,Σ,q0,∂,F)M=(Q, \Sigma, q_0, \partial, F)M=(Q,Σ,q0​,∂,F) where

  • QQQ, Σ\SigmaΣ, q0q_0q0​ and FFF are as in the definition of DFAs;
  • ∂\partial∂ is a transition function that takes ⟨state, input symbol⟩\langle\textrm{state, input symbol}\rangle⟨state, input symbol⟩ pairs and maps each one to a set of states. To say ∂(q,a)={q1,q2,…,qn}\partial(q,a) = \{q_1, q_2, \ldots , q_n\}∂(q,a)={q1​,q2​,…,qn​} means that if the machine is in state qqq and the input symbol aaa is consumed, then the machine may move directly into any one of states q1,q2,…,qnq_1, q_2, \ldots , q_nq1​,q2​,…,qn​. The function ∂\partial∂ must also be defined for every ⟨state,ε⟩\langle\textrm{state},\varepsilon\rangle⟨state,ε⟩ pair. To say ∂(q,ε)={q1,q2,…,qn}\partial(q,\varepsilon) = \{q_1, q_2, \ldots , q_n\}∂(q,ε)={q1​,q2​,…,qn​} means that there are direct ε\varepsilonε-transitions from state qqq to each of q1,q2,…,qnq_1, q_2, \ldots , q_nq1​,q2​,…,qn​.

The formal description of the function ∂\partial∂ is ∂:Q×(Σ∪{ε})→P(Q)\partial : Q \times (\Sigma \cup \{\varepsilon\}) \rightarrow {\mathscr P}(Q)∂:Q×(Σ∪{ε})→P(Q).

The function ∂\partial∂ describes how the machine functions on zero or one input symbol. As with DFAs, we will often want to refer to the behavior of the machine on a string of inputs, and so we use the notation ∂∗(q,w)\partial^*(q,w)∂∗(q,w) as shorthand for "the set of states in which the machine might be if it starts in state qqq and consumes input string www". As with DFAs, ∂∗(q,w)\partial^*(q,w)∂∗(q,w) is determined by the specification of ∂\partial∂. Note that for every state qqq, ∂∗(q,ε)\partial^*(q, \varepsilon)∂∗(q,ε) contains at least qqq, and may contain additional states if there are (sequences of) ε\varepsilonε-transitions out of qqq.

We do have to think a bit carefully about what it means for an NFA to accept a string www. Suppose ∂∗(q0,w)\partial^*(q_0,w)∂∗(q0​,w) contains both accepting and non-accepting states, i.e., the machine could end in an accepting state after consuming www, but it might also end in a non-accepting state. Should we consider the machine to accept www, or should we require every state in ∂∗(q0,w)\partial^*(q_0,w)∂∗(q0​,w) to be accepting before we admit www to the ranks of the accepted? Think of the C++ compiler again: provided that an if statement fits one of the legal syntax specifications, the compiler will accept it. So we take as the definition of acceptance by an NFA: A string www is accepted by an NFA provided that at least one of the states in ∂∗(q0,w)\partial^*(q_0, w)∂∗(q0​,w) is an accepting state. That is, if there is some sequence of steps of the machine that consumes www and leaves the machine in an accepting state, then the machine accepts www. Formally:

Let M=(Q,Σ,q0,∂,F)M= (Q, \Sigma, q_0, \partial, F)M=(Q,Σ,q0​,∂,F) be a nondeterministic finite-state automaton. The string w∈Σ∗w\in\Sigma^*w∈Σ∗ is accepted by MMM iff ∂∗(q0,w)\partial^*(q_0,w)∂∗(q0​,w) contains at least one state qF∈Fq_F \in FqF​∈F.

The language accepted by MMM, denoted L(M)L(M)L(M), is the set of all strings w∈Σ∗w\in\Sigma^*w∈Σ∗ that are accepted by MMM: L(M)={w∈Σ∗ ∣ ∂∗(q0,w)∩F≠∅}L(M) = \{ w \in \Sigma^* \ | \ \partial^*(q_0, w) \cap F \not= \emptyset\}L(M)={w∈Σ∗ ∣ ∂∗(q0​,w)∩F​=∅}.

#Connection with Deterministic Finite-State Automata

It should be fairly clear that every language that is accepted by a DFA is also accepted by an NFA. Pictorially, a DFA looks exactly like an NFA (an NFA that doesn't happen to have any ε\varepsilonε-transitions or multiple same-label transitions from any state), though there is slightly more going on behind the scenes. Formally, given the DFA M=(Q,Σ,q0,δ,F)M=(Q, \Sigma, q_0, \delta, F)M=(Q,Σ,q0​,δ,F), you can build an NFA M′=(Q,Σ,q0,∂,F)M'=(Q, \Sigma, q_0, \partial, F)M′=(Q,Σ,q0​,∂,F) where 4 of the 5 components are the same and where every transition δ(q,a)=q′\delta(q,a) = q'δ(q,a)=q′ has been replaced by ∂(q,a)={q′}\partial(q,a) = \{q'\}∂(q,a)={q′}.

But is the reverse true? Can any NFA-recognized language be recognized by a DFA? Look, for example, at the language in the example above. Can you come up with a DFA that accepts this language? Try it. It's pretty difficult to do. But does that mean that there really is no DFA that accepts the language, or only that we haven't been clever enough to find one?

It turns out that the limitation is in fact in our cleverness, and not in the power of DFAs.1

Theorem: The Subset Construction
Every language that is accepted by an NFA is accepted by a DFA.

Proof: Suppose we are given an NFA N=(P,Σ,p0,∂,Fp)N = (P, \Sigma, p_0, \partial, F_p)N=(P,Σ,p0​,∂,Fp​), and we want to build a DFA D=(Q,Σ,q0,δ,Fq)D=(Q, \Sigma, q_0, \delta, F_q)D=(Q,Σ,q0​,δ,Fq​) that accepts the same language. The idea is to make the states in DDD correspond to subsets of NNN's states, and then to set up DDD's transition function δ\deltaδ so that for any string www, δ∗(q0,w)\delta^*(q_0, w)δ∗(q0​,w) corresponds to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w); i.e., the single state that www gets you to in DDD corresponds to the set of states that www could get you to in NNN. If any of those states is accepting in NNN, www would be accepted by NNN, and so the corresponding state in DDD would be made accepting as well.

So how do we make this work? The first thing to do is to deal with a start state q0q_0q0​ for DDD. If we're going to make this state correspond to a subset of NNN's states, what subset should it be? Well, remember (1) that in any DFA, δ∗(q0,ε)=q0\delta^*(q_0, \varepsilon) = q_0δ∗(q0​,ε)=q0​; and (2) we want to make δ∗(q0,w)\delta^*(q_0, w)δ∗(q0​,w) correspond to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) for every www. Putting these two limitations together tells us that we should make q0q_0q0​ correspond to ∂∗(p0,ε)\partial^*(p_0, \varepsilon)∂∗(p0​,ε). So q0q_0q0​ corresponds to the subset of all of NNN's states that can be reached with no input.

Now we progressively set up DDD's transition function δ\deltaδ by repeatedly doing the following:

  • Find a state qqq that has been added to DDD but whose out-transitions have not yet been added. (Note that q0q_0q0​ initially fits this description.) Remember that the state qqq corresponds to some subset {p1,…,pn}\{p_1, \ldots , p_n\}{p1​,…,pn​} of NNN's states.

  • For each input symbol aaa, look at all NNN's states that can be reached from any one of p1,…,pnp_1, \ldots , p_np1​,…,pn​ by consuming aaa (perhaps making some ε\varepsilonε-transitions as well). That is, look at ∂∗(p1,a)∪…∪∂∗(pn,a)\partial^*(p_1,a) \cup \ldots \cup \partial^*(p_n,a)∂∗(p1​,a)∪…∪∂∗(pn​,a). If there is not already a DFA state q′q'q′ that corresponds to this subset of NNN's states, then add one, and add the transition δ(q,a)=q′\delta(q, a)= q'δ(q,a)=q′ to DDD's transitions.

The above process must halt eventually, as there are only a finite number of states nnn in the NFA, and therefore there can be at most 2n2^n2n states in the DFA, as that is the number of subsets of the NFA's states. The final states of the new DFA are those where at least one of the associated NFA states is an accepting state of the NFA.

Can we now argue that L(D)=L(N)L(D) = L(N)L(D)=L(N)? We can, if we can argue that δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) corresponds to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) for all w∈Σ∗w\in\Sigma^*w∈Σ∗: if this latter property holds, then w∈L(D)w \in L(D)w∈L(D) iff δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) is accepting, which we made be so iff ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) contains an accepting state of NNN, which happens iff NNN accepts www i.e., iff w∈L(N)w \in L(N)w∈L(N).

So can we argue that δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) does in fact correspond to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) for all www? We can, using induction on the length of www.

First, a preliminary observation. Suppose w=xaw=xaw=xa, i.e., www is the string xxx followed by the single symbol aaa. How are ∂∗(p0,x)\partial^*(p_0,x)∂∗(p0​,x) and ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) related? Well, recall that ∂∗(p0,x)\partial^*(p_0,x)∂∗(p0​,x) is the set of all states that NNN can reach when it starts in p0p_0p0​ and consumes xxx: ∂∗(p0,x)={p1,…,pn}\partial^*(p_0,x) = \{p_1, \ldots, p_n\}∂∗(p0​,x)={p1​,…,pn​} for some states p1,…,pnp_1, \ldots, p_np1​,…,pn​. Now, www is just xxx with an additional aaa, so where might NNN end up if it starts in p0p_0p0​ and consumes www? We know that xxx gets NNN to p1p_1p1​ or …\ldots… or pnp_npn​, so xaxaxa gets NNN to any state that can be reached from p1p_1p1​ with an aaa (and maybe some ε\varepsilonε-transitions), and to any state that can be reached from p2p_2p2​ with an aaa (and maybe some ε\varepsilonε-transitions), etc. Thus, our relationship between ∂∗(p0,x)\partial^*(p_0,x)∂∗(p0​,x) and ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) is that if ∂∗(p0,x)={p1,…,pn}\partial^*(p_0,x) = \{p_1, \ldots, p_n\}∂∗(p0​,x)={p1​,…,pn​}, then ∂∗(p0,w)=∂∗(p1,a)∪…∪∂∗(pn,a)\partial^*(p_0,w) = \partial^*(p_1,a) \cup \ldots \cup \partial^*(p_n,a)∂∗(p0​,w)=∂∗(p1​,a)∪…∪∂∗(pn​,a). With this observation in hand, let's proceed to our proof by induction.

We want to prove that δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) corresponds to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) for all w∈Σ∗w\in\Sigma^*w∈Σ∗. We use induction on the length of www.

Base case: Suppose www has length 0. The only string www with length 0 is ε\varepsilonε, so we want to show that δ∗(q0,ε)\delta^*(q_0,\varepsilon)δ∗(q0​,ε) corresponds to ∂∗(p0,ε)\partial^*(p_0,\varepsilon)∂∗(p0​,ε). Well, δ∗(q0,ε)=q0\delta^*(q_0, \varepsilon) = q_0δ∗(q0​,ε)=q0​, since in a DFA, δ∗(q,ε)=q\delta^*(q, \varepsilon) = qδ∗(q,ε)=q for any state~qqq. We explicitly made q0q_0q0​ correspond to ∂∗(p0,ε)\partial^*(p_0,\varepsilon)∂∗(p0​,ε), and so the property holds for www with length 0.

Inductive case: Assume that the desired property holds for some number nnn, i.e., that δ∗(q0,x)\delta^*(q_0,x)δ∗(q0​,x) corresponds to ∂∗(p0,x)\partial^*(p_0,x)∂∗(p0​,x) for all xxx with length nnn. Look at an arbitrary string www with length n+1n+1n+1. We want to show that δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) corresponds to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w). Well, the string www must look like xaxaxa for some string xxx (whose length is nnn) and some symbol aaa. By our inductive hypothesis, we know δ∗(q0,x)\delta^*(q_0,x)δ∗(q0​,x) corresponds to ∂∗(p0,x)\partial^*(p_0,x)∂∗(p0​,x). We know ∂∗(p0,x)\partial^*(p_0,x)∂∗(p0​,x) is a set of NNN's states, say ∂∗(p0,x)={p1,…,pn}\partial^*(p_0,x) = \{p_1, \ldots, p_n\}∂∗(p0​,x)={p1​,…,pn​}.

At this point, our subsequent reasoning might be a bit clearer if we give explicit names to δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) (the state DDD reaches on input www) and δ∗(q0,x)\delta^*(q_0,x)δ∗(q0​,x) (the state DDD reaches on input xxx). Let qw=δ∗(q0,w)q_w = \delta^*(q_0, w)qw​=δ∗(q0​,w), and let qx=δ∗(q0,x)q_x = \delta^*(q_0,x)qx​=δ∗(q0​,x). We know, because w=xaw=xaw=xa, there must be an aaa-transition from qxq_xqx​ to qwq_wqw​. Look at how we added transitions to δ\deltaδ: the fact that there is an aaa-transition from qxq_xqx​ to qwq_wqw​ means that qwq_wqw​ corresponds to the set ∂∗(p1,a)∪…∪∂∗(pn,a)\partial^*(p_1,a) \cup \ldots \cup \partial^*(p_n,a)∂∗(p1​,a)∪…∪∂∗(pn​,a) of NNN's states. By our preliminary observation, ∂∗(p1,a)∪…∪∂∗(pn,a)\partial^*(p_1,a) \cup \ldots \cup \partial^*(p_n,a)∂∗(p1​,a)∪…∪∂∗(pn​,a) is just ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w). So qwq_wqw​ (or δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w)) corresponds to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w), which is what we wanted to prove. Since www was an arbitrary string of length n+1n+1n+1, we have shown that the property holds for n+1n+1n+1.

Altogether, we have shown by induction that δ∗(q0,w)\delta^*(q_0,w)δ∗(q0​,w) corresponds to ∂∗(p0,w)\partial^*(p_0,w)∂∗(p0​,w) for all w∈Σ∗w\in\Sigma^*w∈Σ∗. As indicated at the very beginning of this proof, that is enough to prove that L(D)=L(N)L(D)= L(N)L(D)=L(N). So for any NFA NNN, we can find a DFA DDD that accepts the same language.
