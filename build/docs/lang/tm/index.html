General Grammars and Turing Machines

(Content adapted from Critchlow & Eck)

General Grammars​

At the beginning of this chapter the general idea of a grammar as a set of rewriting or production rules was introduced. For most of the chapter, however, we have restricted our attention to context-free grammars, in which production rules must be of the form A⟶xA\longrightarrow xA⟶x where AAA is a non-terminal symbol. In this section, we will consider general grammars, that is, grammars in which there is no such restriction on the form of production rules. For a general grammar, a production rule has the form u⟶xu\longrightarrow xu⟶x, where uuu is string that can contain both terminal and non-terminal symbols. For convenience, we will assume that uuu contains at least one non-terminal symbol, although even this restriction could be lifted without changing the class of languages that can be generated by grammars. Note that a context-free grammar is, in fact, an example of a general grammar, since production rules in a general grammar are allowed to be of the form A⟶xA\longrightarrow xA⟶x. They just don't have to be of this form. I will use the unmodified term grammar to refer to general grammars.1 The definition of grammar is identical to the definition of context-free grammar, except for the form of the production rules:

A grammar is a 4-tuple (V,Σ,P,S)(V,\Sigma,P,S)(V,Σ,P,S), where:

  • VVV is a finite set of symbols. The elements of VVV are the non-terminal symbols of the grammar.
  • Σ\SigmaΣ is a finite set of symbols such that V∩Σ=∅V\cap\Sigma=\emptysetV∩Σ=∅. The elements of Σ\SigmaΣ are the terminal symbols of the grammar.
  • PPP is a set of production rules. Each rule is of the form u⟶xu\longrightarrow xu⟶x where uuu and xxx are strings in (V∪Σ)∗(V\cup \Sigma)^*(V∪Σ)∗ and uuu contains at least one symbol from VVV.
  • S∈VS\in VS∈V. SSS is the start symbol of the grammar.

Suppose GGG is a grammar. Just as in the context-free case, the language generated by GGG is denoted by L(G)L(G)L(G) and is defined as L(G)={x∈Σ∗  ∣  S⟹G∗x}L(G)=\{x\in\Sigma^*\;|\; S \Longrightarrow_G^* x\}L(G)={x∈Σ∗∣S⟹G∗​x}. That is, a string xxx is in L(G)L(G)L(G) if and only if xxx is a string of terminal symbols and there is a derivation that produces xxx from the start symbol, SSS, in one or more steps.

The natural question is whether there are languages that can be generated by general grammars but that cannot be generated by context-free languages. A simple example of a language that is not context-free is {anbncn  ∣  n∈N}\{a^nb^nc^n\;|\; n\in \N\}{anbncn∣n∈N}. This language can be generated by a general grammar that first generates a string of the form XABCABC⋯ABCXABCABC\cdots ABCXABCABC⋯ABC, where there are nnn groups of ABCABCABC for some n≥0n\geq 0n≥0. Then we use general grammar rules that allow AAA's, BBB's, and CCC's to switch places, until the string looks like XAA⋯ABB⋯BCC⋯CXAA\cdots ABB\cdots BCC\cdots CXAA⋯ABB⋯BCC⋯C. Finally, another set of rules essentially traverse the string from left to right, changing the non-terminals AAA, BBB, and CCC into the corresponding terminals aaa, bbb, and ccc. Here is a grammar that does this:

S⟶SABCS⟶XBA⟶ABCA⟶ACCB⟶BCXA⟶aXX⟶YYB⟶bYY⟶ZZC⟶cZZ⟶ε\begin{aligned} S&\longrightarrow SABC\\ S&\longrightarrow X\\ BA&\longrightarrow AB\\ CA&\longrightarrow AC\\ CB&\longrightarrow BC\\ XA&\longrightarrow aX\\ X&\longrightarrow Y\\ YB&\longrightarrow bY\\ Y&\longrightarrow Z\\ ZC&\longrightarrow cZ\\ Z&\longrightarrow \varepsilon \end{aligned}SSBACACBXAXYBYZCZ​⟶SABC⟶X⟶AB⟶AC⟶BC⟶aX⟶Y⟶bY⟶Z⟶cZ⟶ε​

Here, the first two rules produce one of the strings XXX, XABCXABCXABC, XABCABCXABCABCXABCABC, XABCABCABCXABCABCABCXABCABCABC, and so on. The next three rules allow AAA's to move to the left and CCC's to move to the right, producing a string of the form XAnBnCnXA^nB^nC^nXAnBnCn, for some n∈Nn\in\Nn∈N. The rule XA⟶aXXA\longrightarrow aXXA⟶aX allows the XXX to move through the AAA's from left to right, converting AAA's to aaa's as it goes. After converting the AAA's, the XXX can be transformed into a YYY. The YYY will then move through the BBB's, converting them to bbb's. Then, the YYY is transformed into a ZZZ, which is responsible for converting CCC's to ccc's. Finally, an application of the rule Z⟶εZ\longrightarrow\varepsilonZ⟶ε removes the ZZZ, leaving the string anbncna^nb^nc^nanbncn.

Note that if the rule X⟶YX\longrightarrow YX⟶Y is applied before all the AAA's have been converted to aaa's, then there is no way for the remaining AAA's to be converted to aaa's or otherwise removed from the string. This means that the derivation has entered a dead end, which can never produce a string that consists of terminal symbols only. The only derivations that can produce strings in the language generated by the grammar are derivations in which the XXX moves past all the AAA's, converting them all to aaa's. At this point in the derivation, the string is of the form anXua^nXuanXu where uuu is a string consisting entirely of BBB's and CCC's. Now the rule X⟶YX\longrightarrow YX⟶Y can be applied, producing the string anYua^nYuanYu. Then, if a string of terminal symbols is ever to be produced, the YYY must move past all the BBB's, producing the string anbnYCna^nb^nYC^nanbnYCn. You can see that the use of three separate non-terminals, XXX, YYY, and ZZZ, is essential for forcing the symbols in anbncna^nb^nc^nanbncn into the correct order.

Turing Machines​

We saw hints in the previous section that "computation" is a more general concept than we might have thought. General grammars, which at first encounter don't seem to have much to do with algorithms or computing, turn out to be able to do things that are similar to the tasks carried out by computer programs. In this section, we will see that general grammars are precisely equivalent to computer programs in terms of their computational power, and that both are equivalent to a particularly simple model of computation known as a Turing machine. We shall also see that there are limits to what can be done by computing.

Historically, the theoretical study of computing began before computers existed. One of the early models of computation was developed in the 1930s by the British mathematician, Alan Turing, who was interested in studying the theoretical abilities and limitations of computation. His model for computation is a very simple abstract computing machine which has come to be known as a Turing machine. While Turing machines are not applicable in the same way that regular expressions, finite-state automata, and grammars are applicable, their use as a fundamental model for computation means that every computer scientist should be familiar with them, at least in a general way.

A Turing machine is really not much more complicated than a finite-state automaton or a pushdown automaton.2 Like a FSA, a Turing machine has a finite number of possible states, and it changes from state to state as it computes. However, a Turing machine also has an infinitely long tape that it can use for input and output. The tape extends to infinity in both directions. The tape is divided into cells, which are in one-to-one correspondence with the integers, Z\ZZ. Each cell can either be blank or it can hold a symbol from a specified alphabet. The Turing machine can move back and forth along this tape, reading and writing symbols and changing state. It can read only one cell at a time, and possibly write a new value in that cell. After doing this, it can change state and it can move by one cell either to the left or to the right. This is how the Turing machine computes. To use a Turing machine, you would write some input on its tape, start the machine, and let it compute until it halts. Whatever is written on the tape at that time is the output of the computation.

Although the tape is infinite, only a finite number of cells can be non-blank at any given time. If you don't like the idea of an infinite tape, you can think of a finite tape that can be extended to an arbitrarily large size as the Turing machine computes: If the Turing machine gets to either end of the tape, it will pause and wait politely until you add a new section of tape. In other words, it's not important that the Turing machine have an infinite amount of memory, only that it can use as much memory as it needs for a given computation, up to any arbitrarily large size. In this way, a Turing machine is like a computer that can ask you to buy it a new disk drive whenever it needs more storage space to continue a computation.3

A given Turing machine has a fixed, finite set of states. One of these states is designated as the start state. This is the state in which the Turing machine begins a computation. Another special state is the halt state. The Turing machine's computation ends when it enters its halt state. It is possible that a computation might never end because the machine never enters the halt state. This is analogous to an infinite loop in a computer program.

At each step in its computation, the Turing machine reads the contents of the tape cell where it is located. Depending on its state and the symbol that it reads, the machine writes a symbol (possibly the same symbol) to the cell, moves one cell either to the left or to the right, and (possibly) changes its state. The output symbol, direction of motion, and new state are determined by the current state and the input symbol. Note that either the input symbol, the output symbol, or both, can be blank. A Turing machine has a fixed set of rules that tell it how to compute. Each rule specifies the output symbol, direction of motion, and new state for some combination of current state and input symbol. The machine has a rule for every possible combination of current state and input symbol, except that there are no rules for what happens if the current state is the halt state. Of course, once the machine enters the halt state, its computation is complete and the machine simply stops.

I will use the character # to represent a blank in a way that makes it visible. I will always use hhh to represent the halt state. I will indicate the directions, left and right, with LLL and RRR, so that {L,R}\{L,R\}{L,R} is the set of possible directions of motion. With these conventions, we can give the formal definition of a Turing machine as follows:

A Turing machine is a 4-tuple (Q,Λ,q0,δ)(Q,\Lambda,q_0,\delta)(Q,Λ,q0​,δ), where:

  • QQQ is a finite set of states, including the halt state, hhh.
  • Λ\LambdaΛ is an alphabet which includes the blank symbol, #.
  • q0∈Qq_0\in Qq0​∈Q is the start state.

Our primary interest in Turing machines is as language processors. Suppose that www is a string over an alphabet Σ\SigmaΣ. We will assume that Σ\SigmaΣ does not contain the blank symbol. We can use www as input to a Turing machine M=(Q,Λ,q0,δ)M=(Q,\Lambda,q_0,\delta)M=(Q,Λ,q0​,δ) provided that Σ⊆Λ\Sigma\subseteq\LambdaΣ⊆Λ. To use www as input for MMM, we will write www on MMM's tape and assume that the remainder of the tape is blank. We place the machine on the cell containing the first character of the string, except that if w=εw=\varepsilonw=ε then we simply place the machine on a completely blank tape. Then we start the machine in its initial state, q0q_0q0​, and see what computation it performs. We refer to this setup as "running MMM with input www."

When MMM is run with input www, it is possible that it will just keep running forever without halting. In that case, it doesn't make sense to ask about the output of the computation. Suppose however that MMM does halt on input www. Suppose, furthermore, that when MMM halts, its tape is blank except for a string xxx of non-blank symbols, and that the machine is located on the first character of xxx. In this case, we will say that "MMM halts with output xxx." In addition, if MMM halts with an entirely blank tape, we say that "MMM halts with output ε\varepsilonε." Note that when we run MMM with input www, one of three things can happen: (1) MMM might halt with some string as output; (1) MMM might fail to halt; or (3) MMM might halt in some configuration that doesn't count as outputting any string.

The fact that a Turing machine can produce an output value allows us for the first time to deal with computation of functions. A function f ⁣:A→Bf\colon A\to Bf:A→B takes an input value in the set AAA and produces an output value in the set BBB. If the sets are sets of strings, we can now ask whether the values of the function can be computed by a Turing machine. That is, is there a Turing machine MMM such that, given any string www in the domain of fff as input, MMM will compute as its output the string f(w)f(w)f(w). If this is that case, then we say that fff is a Turing-computable function.

Suppose that Σ\SigmaΣ and Γ\GammaΓ are alphabets that do not contain # and that fff is a function from Σ∗\Sigma^*Σ∗ to Γ∗\Gamma^*Γ∗. We say that fff is Turing-computable if there is a Turing machine M=(Q,Λ,q0,δ)M=(Q,\Lambda,q_0,\delta)M=(Q,Λ,q0​,δ) such that Σ⊆Λ\Sigma\subseteq\LambdaΣ⊆Λ and Γ⊆Λ\Gamma\subseteq\LambdaΓ⊆Λ and for each string w∈Σ∗w\in\Sigma^*w∈Σ∗, when MMM is run with input www, it halts with output f(w)f(w)f(w). In this case, we say that MMM computes the function fff.

We can also use Turing machines to define "computable languages." There are actually two different notions of Turing-computability for languages. One is based on the idea of Turing-computability for functions. Suppose that Σ\SigmaΣ is an alphabet and that L⊆Σ∗L\subseteq\Sigma^*L⊆Σ∗. The characteristic function of LLL is the function χL ⁣:Σ∗→{0,1}\chi_L\colon\Sigma^*\to\{0,1\}χL​:Σ∗→{0,1} defined by the fact that χL(w)=1\chi_L(w)=1χL​(w)=1 if w∈Lw\in Lw∈L and χL(w)=0\chi_L(w)=0χL​(w)=0 if w∉Lw\not\in Lw∈L. Note that given the function χL\chi_LχL​, LLL can be obtained as the set L={w∈Σ∗  ∣  χL(w)=1}L=\{w\in\Sigma^*\;|\; \chi_L(w)=1\}L={w∈Σ∗∣χL​(w)=1}. Given a language LLL, we can ask whether the corresponding function χL\chi_LχL​ is Turing-computable. If so, then we can use a Turing machine to decide whether or not a given string www is in LLL. Just run the machine with input www. It will halt with output χL(w)\chi_L(w)χL​(w). (That is, it will halt and when it does so, the tape will be blank except for a 0 or a 1, and the machine will be positioned on the 0 or 1.) If the machine halts with output 1, then w∈Lw\in Lw∈L. If the machine halts with output 0, then w∉Lw\not\in Lw∈L.

Let Σ\SigmaΣ be an alphabet that does not contain # and let LLL be a language over Σ\SigmaΣ. We say that LLL is Turing-decidable if there is a Turing machine M=(Q,Λ,q0,δ)M=(Q,\Lambda,q_0,\delta)M=(Q,Λ,q0​,δ) such that Σ⊆Λ\Sigma\subseteq\LambdaΣ⊆Λ, {0,1}⊆Λ\{0,1\}\subseteq\Lambda{0,1}⊆Λ, and for each w∈Σ∗w\in\Sigma^*w∈Σ∗, when MMM is run with input www, it halts with output χL(w)\chi_L(w)χL​(w). (That is, it halts with output 0 or 1, and the output is 0 if w∉Lw\not\in Lw∈L and is 1 if w∈Lw\in Lw∈L.) In this case, we say that MMM decides the language LLL.

The second notion of computability for languages is based on the interesting fact that it is possible for a Turing machine to run forever, without ever halting. Whenever we run a Turing machine MMM with input www, we can ask the question, will MMM ever halt or will it run forever? If MMM halts on input www, we will say that MMM "accepts" www. We can then look at all the strings over a given alphabet that are accepted by a given Turing machine. This leads to the notion of Turing-acceptable languages.

Let Σ\SigmaΣ be an alphabet that does not contain #, and let LLL be a language over Σ\SigmaΣ. We say that LLL is Turing-acceptable if there is a Turing machine M=(Q,Λ,q0,δ)M=(Q,\Lambda,q_0,\delta)M=(Q,Λ,q0​,δ) such that Σ⊆Λ\Sigma\subseteq\LambdaΣ⊆Λ, and for each w∈Σ∗w\in\Sigma^*w∈Σ∗, MMM halts on input www if and only if w∈Lw\in Lw∈L. In this case, we say that MMM accepts the language LLL.

It should be clear that any Turing-decidable language is Turing-acceptable. In fact, if LLL is a language over an alphabet Σ\SigmaΣ, and if MMM is a Turing machine that decides LLL, then it is easy to modify MMM to produce a Turing machine that accepts LLL. At the point where MMM enters the halt state with output 0, the new machine should enter a new state in which it simply moves to the right forever, without ever halting. Given an input w∈Σ∗w\in\Sigma^*w∈Σ∗, the modified machine will halt if and only if MMM halts with output 1, that is, if and only if w∈Lw\in Lw∈L.
